# Crucix Cycle 057 - 2026-04-24

## Summary
- Completed Epic 17 effort 17.2 by making degraded macro-card states operator-readable instead of silently blank.
- Macro cards now distinguish healthy values from setup debt, external limits, stale/degraded states, and generic unavailability by using source-health truth from `/api/data.health`.

## What changed
- Updated `dashboard/public/jarvis.html`.
- Added `healthByName` lookup inside `renderLower()` based on `D.health`.
- Added `sourceStateText(name, fallback)` helper to translate health states into short operator-facing labels:
  - `Not configured`
  - `Rate limited`
  - `Stale source`
  - `Degraded source` / specific operator label
  - `No current data`
  - fallback `Unavailable`
- Wired those source-state labels into macro cards so missing values no longer appear as unexplained blanks.
- Specifically applied explicit source-state handling to:
  - `Fed Funds` via `FRED`
  - `CPI MoM` via `BLS`
  - `Unemployment` via `BLS`
  - `VIX` fallback semantics via `YFinance`
  - `GSCPI` fallback semantics via `GSCPI`
