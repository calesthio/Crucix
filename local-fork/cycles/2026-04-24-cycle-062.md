# Crucix dev cycle 062

- Date: 2026-04-24
- Plan item: Epic 19, effort 19.1
- Title: Convert cluster-review quality telemetry into a bounded actionable operator queue instead of metrics-only visibility

## What changed

- Added shared server helper `buildOperatorReviewQueue(review, { maxItems })`.
- The queue converts review telemetry into a bounded operator surface with:
  - total active items
  - visible item count
  - max item cap
  - has-more indicator
  - summary text
  - top repeated reasons
  - compact operator-facing queue items with region, severity, item count, chronic flag, and suggested action
- Updated `/api/brief/news/review` to return both raw decorated `review` data and the new bounded `queue` object.
- Updated `/api/data` to return `reviewQueue` for the main dashboard payload.
- Normalized `/api/data` and the review endpoint to return an empty bounded queue object instead of `null` when there are no active review items.
- Added a new dashboard right-rail `Review Queue` panel that renders the bounded queue and empty state.
- Restarted the local Crucix runtime so the live service on `127.0.0.1:3117` serves the new queue payload and dashboard panel.
