## Goal
Add preset-specific density tuning and alternate topbar presentation so operator, diagnostics, source-ops, and wallboard modes can each use different layout pressure budgets instead of sharing one global desktop compromise.

## Backlog review and promotions

## Changes shipped
- Extended built-in workspace preset definitions in both runtime and dashboard fallback paths with:
  - `densityMode`
  - `topbarMode`
- Wired the served dashboard to apply those preset presentation modes through body dataset attributes:
  - `data-density-mode`
  - `data-topbar-mode`
- Added density-specific CSS tuning so dense desktop presets reduce map height and side-rail width pressure.
- Added compact and briefing topbar variants so diagnostics/source-ops can use tighter controls while wallboard mode gets a more broadcast-friendly topbar arrangement.
- Preset metadata now flows through operator/admin settings payloads and dashboard bootstrap state, so runtime contracts stay honest about which presentation modes each preset uses.
- Extended regression coverage across served-page smoke, operator settings persistence, settings contract, and UI payload contract tests.
