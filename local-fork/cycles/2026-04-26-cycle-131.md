# Crucix Dev Cycle 131 - Checked service definitions for launchd and systemd

- Date: 2026-04-26
- Plan item: `epic-33 / effort-33.6`
- Status: completed

## Goal
Provide checked example `launchd` and `systemd` service definitions plus restart-policy guidance so Epic 33 hardening is easier to operationalize on real hosts.

## Changes shipped
- Added `deploy/launchd/com.crucix.local.plist`:
  - launches Crucix with `node server.mjs`
  - sets `CRUCIX_AUTO_OPEN_BROWSER=0` for headless service mode
  - enables `RunAtLoad`, bounded restart behavior via `KeepAlive`, and repo-local stdout/stderr logs
- Added `deploy/systemd/crucix.service`:
  - uses `ExecStartPre=/usr/bin/env node --check server.mjs`
  - uses `ExecStart=/usr/bin/env node server.mjs`
  - sets `Restart=on-failure`, `RestartSec=10`, `KillSignal=SIGTERM`, and `TimeoutStopSec=30`
- Added `docs/deployment-service-examples.md`:
  - install and management commands for both service managers
  - placeholder paths to replace before install
  - restart-policy guidance tied to Crucix health and graceful shutdown semantics
  - post-install `/api/health` validation guidance
- Updated `README.md` to point operators at the new deployment examples.
- Added `test/deployment-service-examples.test.mjs` to keep the example files and docs from drifting.
