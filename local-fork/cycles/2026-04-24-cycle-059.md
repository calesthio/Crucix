# Crucix dev cycle 059

- Date: 2026-04-24
- Plan item: Epic 18, effort 18.1
- Title: Unify operator-visible LLM status language across analysis, leverageable ideas, and review surfaces so off, fallback, configured, and parse-failed states do not contradict each other

## What changed

- Replaced contradictory dashboard LLM labels with a small operator-facing vocabulary:
  - `LLM APPLIED`
  - `LLM FALLBACK`
  - `LLM PENDING`
  - `LLM UNAVAILABLE`
- Updated the Agent Analysis panel badge and metadata line to describe whether LLM refinement was applied, unavailable, queued, or attempted but rejected in favor of deterministic fallback.
- Updated the right-rail Analysis Review badge and explanation line to use the same vocabulary.
- Updated Leverageable Ideas to use the same LLM status language instead of mixed labels like `AI ENHANCED`, `LLM OFF`, or generic `PENDING`.
- Updated compact iMessage brief wording for news clustering so mixed LLM/heuristic runs report as `LLM partial fallback` instead of the vaguer `heuristic fallback` phrasing.
- Added regression coverage for dashboard status coherence and iMessage brief LLM wording.
