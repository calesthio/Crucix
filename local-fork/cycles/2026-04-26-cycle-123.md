# Crucix Dev Cycle 123 - Startup and deployment hardening

- Date: 2026-04-26
- Plan item: `epic-33 / effort-33.1`
- Status: completed

## Goal
Add startup and deployment hardening with environment validation, secret sanity checks, graceful shutdown, and process-manager-friendly health semantics.

## Changes shipped
- Updated `server.mjs`:
  - added boot-time environment validation with fatal issues versus warnings
  - added secret sanity checks for incomplete or placeholder config
  - added headless-safe browser auto-open control via `CRUCIX_AUTO_OPEN_BROWSER`
  - added runtime lifecycle phases: `booting`, `serving`, `shutting-down`, `stopped`
  - hardened `/api/health` to expose:
    - `lifecycle`
    - `startupValidation`
    - `503` status during shutdown for process-manager-friendly health semantics
  - added graceful shutdown handling for `SIGTERM` and `SIGINT`
    - clears sweep and watchdog intervals
    - stops Telegram polling
    - stops Discord client
    - closes SSE clients
    - closes the HTTP server cleanly
- Added `test/startup-hardening.test.mjs`:
  - verifies health lifecycle and startup validation on a booted server
  - verifies clean `SIGTERM` exit
  - verifies fail-fast behavior for incomplete secret configuration
- Updated `.env.example`:
  - documented `CRUCIX_AUTO_OPEN_BROWSER`
- Updated `local-fork/BACKLOG.md`:
  - added a follow-on recommendation for checked `launchd` or `systemd` service definitions plus restart-policy guidance
