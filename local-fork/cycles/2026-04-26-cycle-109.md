# Crucix Dev Cycle 109 - Active LLM provider readiness heartbeat

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.6`
- Status: completed

## Goal
Add an active LLM provider readiness probe plus last-success and last-failure heartbeat so `/llm-ops` health reflects live reachability and model readiness rather than only static configuration posture.

## Changes shipped
- Added a shared default `probe()` method to `lib/llm/provider.mjs` that performs a tiny bounded completion-based readiness check.
- Added cached provider-readiness heartbeat state in `server.mjs`, including:
  - active probe status
  - last checked timestamp
  - last success details
  - last failure details
  - consecutive failure count
  - probe latency
  - staleness and in-flight state
- Updated `/api/llm/operations` so it refreshes the readiness probe before returning the LLM operations contract.
- Updated `/api/health` to expose `llmProviderReadiness` and opportunistically refresh stale heartbeat state in the background.
- Updated the LLM operations contract so `provider.health` now reflects active readiness results, and `provider.readiness` carries the heartbeat details.
- Updated `dashboard/public/llm-ops.html` to render a `Provider readiness heartbeat` card and probe summary.
- Expanded regression coverage in:
  - `test/settings-contract.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
