## Goal
Add configurable alert routing and escalation for stale sweeps, source failures, elevated review pressure, and degraded inference paths.

## Changes shipped
- Updated `crucix.config.mjs`
  - added runtime alerting config:
    - `alerting.enabled`
    - `alerting.cooldownMinutes`
    - `alerting.escalationCooldownMinutes`
- Updated `server.mjs`
  - added operator settings defaults and normalization for:
    - operational alert enablement
    - default route and escalation route
    - per-policy thresholds, cooldowns, and escalation counts
  - added `OPERATIONAL_ALERTS_STATE_KEY`
  - added `summarizeOperationalAlertState()` to synthesize operator-visible alert posture for:
    - stale sweep
    - source failures
    - review pressure
    - inference degraded
  - added `dispatchOperationalAlert()` route dispatch helper
  - added `processOperationalAlerts()` cooldown and escalation state machine persisted through memory signal state
  - exposed operational alert state on:
    - `/api/health`
    - `/api/settings`
  - invoked operational alert processing after sweep, ideas enrichment, and analysis enrichment publishes
- Updated tests:
  - `test/operator-settings-persistence.test.mjs`

  - `test/settings-endpoint-contract.test.mjs`
