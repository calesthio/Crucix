# Crucix Dev Cycle 111 - Review workflow actions and audit trail

- Date: 2026-04-26
- Plan item: `epic-32 / effort-32.1`
- Status: completed

## Goal
Turn review queues and low-confidence diagnostics into direct operator workflows with acknowledge, snooze, suppress, promote, and quarantine actions, plus attributable audit capture.

## Changes shipped
- Extended the review queue and runtime contracts with `reviewWorkflow.version = "review-workflow-v1"`, direct action metadata, low-confidence source recommendations, shadow-source promote candidates, and recent audit-trail summaries.
- Added local-admin workflow endpoints:
  - `POST /api/review-workflow/action`
  - `GET /api/review-workflow/audit`
- Added persisted workflow audit capture in `runs/review-workflow-audit.json` for:
  - `ack`
  - `snooze`
  - `suppress-source`
  - `quarantine-source`
  - `promote-shadow`
- Reused existing review acknowledgements and source-control persistence instead of inventing a second workflow-state system.
- Updated `/api/data` and `/api/brief/news/review` to publish the workflow contract to operator surfaces.
- Updated `/diagnostics.html` to render:
  - direct review workflow buttons for low-confidence sources and shadow promote candidates
  - recent workflow audit entries
  - direct links to the new audit endpoint
- Extended queue regression coverage so actionable review items keep their direct operator workflow affordances.
