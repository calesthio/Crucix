# Crucix dev cycle 077

- Date: 2026-04-25
- Plan item: Epic 25, effort 25.2
- Title: Define policy-driven lifecycle transitions that can auto-advance candidates through pre-production stages up to shadow while requiring human approval before active promotion

## What changed

- Added an explicit lifecycle transition contract at `source-ops/lifecycle-transition-policy.json` defining:
  - default entry state
  - allowed next states for each lifecycle stage
  - which states agents may auto-advance
  - the pre-production auto-advance ceiling of `shadow`
  - promotion-readiness guards for `shadow`, `approved`, and `active`
  - operator expectations for shadow, approved, and active boundaries
- Extended `source-ops/profile.json` with:
  - `lifecycleTransitionPolicyPath`
  - `transitionPolicy.defaultEntryState`
  - `transitionPolicy.agentMayAutoAdvanceTo`
  - `transitionPolicy.humanApprovalRequiredFor`
  - `transitionPolicy.policySource`
- Extended `source-ops/schemas/task-packet.schema.json` so task packets must now carry:
  - `policy.preProductionAutoAdvanceMax`
  - `policy.lifecycleTransitionPolicyPath`
- Updated all bounded source-ops example task packets to include the new lifecycle-transition policy fields.
- Extended `source-ops/source-registry.schema.json` and the canonical shadow example in `source-ops/source-registry.seed.json` so shadow candidates can declare explicit `eligibleNextStates`.
- Extended `lib/source-ops-runtime.mjs` so the operator-facing `sourceOps` surface now includes:
  - `contract.lifecycleTransitionPolicyPath`
  - `contract.transitionPolicy`
  - `lifecycleTransitions.version`
  - `lifecycleTransitions.defaultEntryState`
  - `lifecycleTransitions.preProductionAutoAdvanceMax`
  - `lifecycleTransitions.activePromotionRequiresHumanApproval`
  - summarized per-state transition metadata
  - `shadow.items[].eligibleNextStates`
- Expanded validation coverage in:
  - `test/source-ops-contract.test.mjs`
  - `test/source-ops-runtime.test.mjs`
