# Crucix dev cycle 076

- Date: 2026-04-25

- Title: Add shadow-state handling so candidate sources can be observed and scored before they influence production operator output

## What changed

- Extended `source-ops/profile.json` with an explicit `shadowPolicy` block so the contract states that:
  - shadow candidates are blocked from production influence
  - the shadow operator surface is source-ops-only
  - scoring is required before a source can remain in shadow
  - `shadow-ready` is the minimum readiness target before human review
- Extended `source-ops/source-registry.schema.json` with optional structured `shadow` metadata for pre-production candidates, including:
  - production block flag
  - observation mode
  - scorecard and overlap assessment references
  - promotion readiness
  - shadow notes and observation timestamp
- Added a canonical example shadow candidate to `source-ops/source-registry.seed.json`:
  - lifecycle `shadow`
  - `enabledByDefault: false`
  - explicit references to the grading and overlap artifacts
  - review provenance showing agent-reviewed pre-production evidence
- Extended `lib/source-ops-runtime.mjs` so the operator-facing `sourceOps` surface now includes:
  - `contract.shadowPolicy`
  - `inventory.productionActive`
  - `inventory.preProduction`
  - `shadow.total`
  - `shadow.readyForHumanReview`
  - `shadow.blockedFromProduction`
  - `shadow.items[]` summaries with readiness and artifact references
- Expanded contract and runtime coverage in:
  - `test/source-ops-contract.test.mjs`
  - `test/source-ops-runtime.test.mjs`
