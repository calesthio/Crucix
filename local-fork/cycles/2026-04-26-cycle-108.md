# Crucix Dev Cycle 108 - Deferred ideas watchdog classification

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.5`
- Status: completed

## Goal
Extend runtime phase telemetry and watchdog recovery classification to deferred ideas enrichment so `/api/health` and the watchdog can distinguish post-publish idea-generation hangs from deferred agent-analysis hangs.

## Changes shipped
- Extended `getSweepWatchdogSnapshot()` in `server.mjs` so deferred phases remain watchdog-active even after the main sweep has published.
- Added explicit deferred ideas recovery classification:
  - `deferred-ideas-hang`
  - existing `deferred-analysis-hang` remains intact
- Extended `recoverHungSweep()` so watchdog recovery now also clears deferred ideas or deferred analysis jobs, marks the affected runtime job as `watchdog-recovered`, and increments cancellation telemetry.
- Adjusted sweep cleanup so deferred `llm-ideas-refinement` and `llm-analysis-refinement` phases are not prematurely forced back to completed during sweep teardown.
- Tightened synthesis phase completion so it only auto-completes when synthesis is still the active phase.
- Expanded watchdog regression coverage in `test/sweep-watchdog.test.mjs` for overdue deferred ideas refinement.
- Updated lifecycle harness coverage in `test/sweep-lifecycle-transitions.test.mjs` so the extracted sweep runner still exercises the live source-ops and memory hooks used by `runSweepCycle()`.
