# Crucix Cycle 053 - 2026-04-24

## Summary
- Completed Epic 16 effort 16.1 by making globe/3D failure degrade cleanly to flat-mode instead of leaving a misleading broken primary control.
- The operator-visible fix is that once 3D is unavailable or globe init fails, the dashboard stops advertising globe mode as a usable interaction and clearly remains in flat fallback.

## What changed
- Updated `dashboard/public/jarvis.html`.
- Added explicit 3D availability tracking:
  - `globeFailed`
  - `globeFailureReason`
  - `canUse3D()`
  - `updateMapModeControls()`
  - `markGlobeUnavailable()`
- `initMap()` now checks 3D availability before attempting globe startup.
- `initGlobe()` now fails fast when `Globe`/`THREE` are unavailable or the UI has already marked globe mode failed.
- `toggleMapMode()` now stops retry-loop behavior and uses bounded fallback logic:
  - if 3D is unavailable, it immediately settles into flat mode
  - the map-mode button becomes disabled and relabeled to `3D UNAVAILABLE`
  - the map hint switches to explicit flat fallback messaging instead of implying globe mode is still available
- Added a disabled visual state for the map-mode control.
