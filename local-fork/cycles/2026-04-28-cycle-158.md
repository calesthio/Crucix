## Changes shipped
### Server-side concurrency contract
- Extended operator settings persistence with monotonic `revision` state and revision-derived ETags.
- Added helpers to advertise concurrency state, resolve `If-Match` or body-provided revision tokens, and return structured `428` / `409` conflict payloads.
- Enforced concurrency tokens for:
  - `PUT /api/settings/operator`
  - `POST /api/settings/import`
  - source-control writes that persist through operator settings:
    - `POST /api/source-ops/control`
    - source-control actions triggered through `POST /api/review-workflow/action`
- Updated admin and operator contracts to expose revision, ETag, concurrency requirements, and the write header contract.

### Admin and operator surfaces
- Updated `dashboard/public/admin-settings.html` so save/import flows send the current concurrency token, refresh local state after successful writes, and surface API-provided conflict detail instead of generic HTTP failures.
- Updated `dashboard/public/source-ops.html` so local source-control actions fetch and reuse admin concurrency state before persisting suppress or quarantine changes.
- Updated `dashboard/public/diagnostics.html` so review-workflow source actions also carry the current concurrency token.

### Test coverage
- Expanded endpoint and persistence tests to assert revision and ETag exposure, write-concurrency capability flags, and the `If-Match` admin contract.
- Added stale-write conflict coverage to verify old tokens fail with a bounded conflict instead of silently winning.
- Updated source-control and review-workflow write-path tests to use the advertised concurrency token flow.
