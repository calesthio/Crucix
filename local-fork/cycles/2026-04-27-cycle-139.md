## Goal
Add browser-level regression coverage for the shared operator surface shell so active-nav state, boundary copy, and admin action availability are validated after client-side render instead of only by static HTML fetches.

## Changes shipped
- Added `test/ops-shell-browser-contract.test.mjs`.
- The new test boots a real Crucix server and uses the local `agent-browser` CLI to validate browser-rendered shared-shell behavior on:
  - `/settings`
  - `/source-ops`
  - `/diagnostics`
  - `/llm-ops`
  - `/admin/settings`
- The test verifies after client-side render that:
  - shared navigation mounts with the full six-surface shell
  - the correct nav card is active for each surface
  - operator-facing boundary copy remains present on `/settings`
  - admin-only controls are absent from `/settings`
  - admin-only controls are present on `/admin/settings`
  - key shared-shell labels and page-specific copy remain visible on the rendered surface
- Updated `local-fork/plan.json` to mark `effort-34.7` complete and advance local cycle tracking to cycle 139.
