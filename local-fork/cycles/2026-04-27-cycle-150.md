# Crucix Dev Cycle 150 - Added retained restart-safe audit visibility to runtime diagnostics

- Date: 2026-04-27
- Plan item: `epic-41 / effort-41.1`
- Status: completed

## Goal
Promote the retained restart-audit backlog item into the roadmap and expose restart-safe outcome history plus PID-rotation proof in operator-visible runtime diagnostics, instead of relying on shell output alone.

## Changes shipped
- Promoted restart-audit backlog work into `local-fork/plan.json` as `epic-41 / effort-41.1` and completed it in the same cycle.
- Marked Epics 39 and 40 complete in the dev plan, since all scheduled efforts under those epics are now done.
- Added retained restart audit support in `lib/runtime-restart.mjs`:
  - `runtimeRestartAuditDefaults()`
  - `loadRuntimeRestartAudit(path)`
  - `appendRuntimeRestartAudit(path, entry, maxEntries)`
- Added `RUNTIME_RESTART_AUDIT_PATH` and local audit-loading support in `server.mjs`.
- Extended `buildRuntimeControlContract(snapshot)` so admin/runtime diagnostics now expose:
  - `restartAudit.version = "runtime-restart-audit-v1"`
  - `restartAudit.updatedAt`
  - `restartAudit.totalEntries`
  - `restartAudit.recentEntries`
  - contract notes explaining the evidence source and meaning
- Updated `POST /api/runtime/control` so `restart-safe` writes retained queue and queue-failure audit entries.
- Updated `scripts/restart-helper.mjs` so successful restart-safe runs append completed audit entries including:
  - previous PIDs
  - started PID
  - live PID
  - replacement mode
  - rotation proof
  - health status
  - last sweep snapshot value
- Restart-helper failures now append explicit failed audit entries before exiting.
- Added regression coverage for retained restart audit storage and admin/runtime contract exposure.
