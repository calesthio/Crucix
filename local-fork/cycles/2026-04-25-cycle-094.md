# Crucix Dev Cycle 094 - Separate operator, diagnostics, and admin surfaces

- Date: 2026-04-25
- Plan item: `epic-28 / effort-28.4`
- Status: completed

## Goal
Separate live operator views, diagnostics, and administration into clearer information architecture so the dashboard stops mixing operations with debug clutter.

## Changes shipped
- Extended injected dashboard runtime metadata with dedicated URLs for:
  - `/settings`
  - `/diagnostics`
  - `/admin/settings`
- Added a dedicated local-only diagnostics route and page at `/diagnostics`.
- Updated the main dashboard topbar so operator, diagnostics, and admin actions are explicit instead of a single ambiguous settings button.
- Updated the operator settings contract to expose `access.diagnosticsSurface` alongside the admin surface.
- Updated the admin contract to expose both operator and diagnostics surfaces.
- Added cross-links among operator settings, diagnostics, and admin settings pages so the three surfaces behave like separate but connected views.
- Created a diagnostics page focused on:
  - runtime/review diagnostics
  - review pressure and acknowledgement posture
  - analysis review state
  - direct debug endpoint links

## Validation
- Contract/runtime tests passed:
  - `node --test test/settings-contract.test.mjs test/settings-endpoint-contract.test.mjs test/operator-settings-persistence.test.mjs test/runtime-config-contract.test.mjs`
- Live runtime verification on `http://127.0.0.1:3117` confirmed:
  - `operatorDiagnosticsSurface = /diagnostics`
  - dashboard serves explicit diagnostics and admin actions
  - diagnostics page title renders correctly
  - diagnostics page links to admin settings
