# Crucix Dev Cycle 124 - SQLite runtime history persistence

- Date: 2026-04-26
- Plan item: `epic-33 / effort-33.2`
- Status: completed

## Goal
Persist runtime history for source health, review pressure, LLM failures, and performance trends in a proper local datastore instead of only transient in-memory surfaces.

## Changes shipped
- Added `lib/delta/runtime-history-store.mjs`:
  - SQLite-backed local runtime history store using `node:sqlite`
  - persists compact runtime runs and deltas
  - persists signal-state blobs previously kept only in JSON-backed memory state
- Updated `lib/delta/memory.mjs`:
  - initializes SQLite store at `runs/runtime-history.sqlite`
  - writes each compact run into SQLite on `addRun()`
  - reads and writes `getSignalState()` / `setSignalState()` through SQLite first
  - reconstructs retained historical views from persisted SQLite run history with file fallback compatibility
  - added:
    - `getSourceHealthHistory()`
    - `getReviewPressureHistory()`
    - `getLlmFailureHistory()`
- Updated `server.mjs`:
  - adds persisted histories onto synthesized runtime data:
    - `sourceHealthHistory`
    - `reviewPressureHistory`
    - `llmFailureHistory`
    - existing `sourcePerformanceHistory` remains
- Added `test/runtime-history-store.test.mjs`
  - verifies SQLite-backed persistence and reload behavior
- Updated `local-fork/BACKLOG.md`
  - added follow-on recommendation for datastore diagnostics and export visibility
