# Crucix dev cycle 088 - runtime config contract and drift reporting

## Plan target
- Epic 27, effort 27.2
- Define a versioned Crucix runtime configuration contract with schema validation, sane defaults, effective-config reporting, and boot-time drift checks across file, env, and live runtime state.

## What changed
- Added `buildRuntimeConfigContract()` in `server.mjs` and exposed its output through the operator settings surface.
- Introduced a versioned runtime config contract `runtime-config-v1` with:
  - built-in defaults
  - effective runtime values
  - schema metadata and invariants
  - validation issues and warnings
  - env override provenance
  - drift summary across default vs effective values
  - redacted handling for sensitive secrets
- Extended `/api/settings` so the operator settings control plane now includes a dedicated `config` section with contract, validation, and drift summary.
- Extended `dashboard/public/settings.html` so the settings UI now renders runtime config contract state, validation state, drift count, and env-override counts.

## Validation
- Focused suite passed:
  - `node --test test/settings-contract.test.mjs test/settings-endpoint-contract.test.mjs test/runtime-config-contract.test.mjs test/freshness-policy-runtime-override.test.mjs test/runtime-llm-status.test.mjs`
- Live runtime proof on port `3117` after listener refresh:
  - `/api/settings` returned `config.contract.version: runtime-config-v1`
  - live config contract reported `refresh: 15`, `debug: local-only`, `valid: true`, and drift summary with total, drifted, env override, and defaulted entry counts
  - `/settings` rendered the new `Runtime config` section in the operator settings UI
