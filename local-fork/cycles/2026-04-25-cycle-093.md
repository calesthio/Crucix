# Crucix Dev Cycle 093 - Saved workspace presets

- Date: 2026-04-25
- Plan item: `epic-28 / effort-28.3`
- Status: completed

## Goal
Add saved workspace presets with quick switching so operators can move between distinct dashboard working modes without manually re-tuning panel state each time.

## Changes shipped
- Extended operator settings persistence with `preferences.layout.workspacePreset`.
- Normalized and validated the workspace preset field in the server-side settings store.
- Upgraded the operator settings contract so layout controls now expose:
  - `workspacePreset`
  - `availableWorkspacePresets`
- Updated layout mutability reporting so presets are represented as persisted server-file state instead of planned-only metadata.
- Added dashboard preset definitions and quick switching for:
  - `operator`
  - `diagnostics`
  - `source-ops`
  - `executive-briefing`
- Applied preset bundles across:
  - display mode
  - map mode
  - visuals mode
  - default region
  - active layer
  - panel state bundle
- Added a topbar preset strip and current preset badge so active workspace mode is visible and switchable in the main dashboard.

## Validation
- Contract/runtime tests passed:
  - `node --test test/settings-contract.test.mjs test/settings-endpoint-contract.test.mjs test/operator-settings-persistence.test.mjs test/runtime-config-contract.test.mjs`
- Live runtime verification on `http://127.0.0.1:3117` confirmed:
  - `layout.current = operator`
  - `layout.controls.workspacePreset = operator`
  - available presets are `operator`, `diagnostics`, `source-ops`, and `executive-briefing`
  - served dashboard HTML includes `renderWorkspacePresetStrip`, `applyWorkspacePreset`, and `preset-btn`

## Follow-on recommendation added to backlog
- Add operator-editable preset authoring, clone/reset controls, and import-export so workspace presets stop being code-defined bundles and can be tuned safely from the UI.
