# Crucix Dev Cycle 116 - Attribution diagnostics aggregator classification

- Date: 2026-04-26
- Plan item: `epic-32 / effort-32.6`
- Status: completed

## Goal
Distinguish expected multi-publisher runtime buckets from true alias-collision anomalies in attribution diagnostics, so broad aggregators like `GDELT` do not over-report routine publisher diversity as operator-actionable alias risk.

## Changes shipped
- Updated `lib/source-ops-runtime.mjs`:
  - added runtime-bucket classification to separate expected multi-publisher aggregators from true alias-collision anomalies
  - moved GDELT-style buckets into `expectedMultiPublisherBuckets` instead of `aliasCollisions`
  - added `summary.expectedMultiPublisherBucketCount`
  - kept double-count risk focused on true alias-like collisions while still surfacing expected aggregator buckets as context
- Updated `dashboard/public/source-ops.html`:
  - attribution diagnostics now separately show alias collisions and expected multi-publisher buckets
  - operator-facing copy now makes alias risk versus normal aggregator behavior clearer
- Extended `test/source-ops-runtime.test.mjs`:
  - `GDELT` is treated as an expected multi-publisher bucket
  - a true alias-style case (`Operator Feed`) is still flagged as a collision
  - assertions now confirm GDELT no longer inflates alias-collision reporting
