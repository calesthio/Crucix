# Crucix Dev Cycle 152 - Added poll-backed restart-safe completion feedback on ops surfaces

- Date: 2026-04-27
- Plan item: `epic-43 / effort-43.1`
- Status: completed

## Goal
Promote the queued restart-safe live-refresh backlog item into the roadmap and add simple poll-backed completion feedback so admin and diagnostics surfaces no longer require manual reloads to notice when a queued restart-safe action finishes.

## Changes shipped
- Promoted restart-audit live-refresh work into `local-fork/plan.json` as `epic-43 / effort-43.1` and completed it in the same cycle.
- Updated `dashboard/public/admin-settings.html`:
  - added inline restart-audit poll scheduling helpers
  - after `Restart` is pressed, the page now reports `Queued safe runtime restart, polling for completion…`
  - if the latest retained restart-audit entry is still `queued`, the page automatically reboots its own data fetch after a short delay
  - when a non-queued outcome appears, status text is updated with the latest restart-safe phase and status
- Updated `dashboard/public/diagnostics.html`:
  - added matching poll-backed restart-audit refresh helpers
  - when diagnostics sees the latest restart-audit entry is `queued`, it automatically refreshes after a short delay and reports `Restart-safe is queued, polling diagnostics…`
  - when the queued action resolves, status text switches to the latest observed outcome
- Added served-page regression checks proving both surfaces now contain the polling feedback language.
- Advanced cycle tracking to 152.
