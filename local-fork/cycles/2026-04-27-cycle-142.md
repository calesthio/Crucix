# Crucix Dev Cycle 142 - Added critical-event promotion states and evidence summaries

- Date: 2026-04-27
- Plan item: `epic-35 / effort-35.3`
- Status: completed

## Goal
Add explicit critical-event promotion rules and operator-visible confidence states so the urgent queue explains when a candidate is still preliminary, when it has crossed the paging threshold, and what evidence basis or unresolved gaps drove that state.

## Changes shipped
- Extended the critical-event queue contract so each candidate now exposes:
  - `promotion.state`
  - `promotion.crossedThreshold`
  - `promotion.crossedAt`
  - `promotion.basis`
  - `promotion.unresolved`
  - `promotion.summary`
- Added `evidenceSummary` per candidate with bounded source/reason excerpts plus trust-tier and hit counts.
- Recorded `promotedAt` for candidates that first cross threshold.
- Added queue-level `confidenceStates` enumeration so consumers can treat the confidence labels as a contract rather than implied strings.
- Preserved current queue behavior for monitoring, retry, promotion, and discard, but made the threshold evaluation explain itself explicitly.
- Updated test coverage for the new queue contract fields on settings and UI-facing payloads.
- Updated `local-fork/plan.json` to mark `effort-35.3` complete and advance local cycle tracking to 142.
