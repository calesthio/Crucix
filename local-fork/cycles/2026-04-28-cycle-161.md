# Crucix Dev Cycle 161 - Added saved preset/reset controls for operator panel layouts

- Date: 2026-04-28
- Plan item: `epic-36 / effort-36.4`
- Status: completed

## Goal
Add saved panel-layout reset support plus named operator presets so a bad panel state can be recovered intentionally and operators can switch between analyst, narrow-screen, source-ops, and wallboard profiles without guessing what each preset means.

## Changes shipped
### Shared preset contract
- Extended the settings contract with named workspace preset metadata, including labels, descriptions, profile tags, and panel defaults.
- Added contract fields for:
  - `layout.controls.currentWorkspacePresetLabel`
  - `layout.controls.namedPresets`
  - `layout.controls.resetSupported`
  - `layout.controls.resetActions`
- Renamed the user-facing preset labels to clearer intent-driven names:
  - `operator` -> `Analyst`
  - `diagnostics` -> `Narrow Screen`
  - `executive-briefing` -> `Wallboard`

### Dashboard preset/reset UX
- Updated `dashboard/public/jarvis.html` to consume the runtime-injected preset library instead of relying only on local hardcoded labels.
- The topbar preset strip now exposes clearer labels and a `Reset Layout` action that restores the current preset defaults.
- Runtime bootstrap now injects `workspacePresetLibrary` so the dashboard can stay aligned with the server contract.

### Local admin save/reset flow
- Updated `dashboard/public/admin-settings.html` so the admin surface now persists `workspacePreset` alongside the other layout controls.
- Added preset workflow controls to the admin surface:
  - load preset into the form
  - reset panels to the selected preset
  - clear saved panel overrides
- These controls use the existing local-admin nonce and concurrency boundary, so the reset is actually saved through the supported write path instead of pretending the public dashboard can always persist it.

### Read-only settings visibility
- Updated `dashboard/public/settings.html` so the operator-facing read-only settings surface shows the active preset label, reset support, and the named preset catalog.
