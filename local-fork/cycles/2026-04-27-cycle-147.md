# Crucix Dev Cycle 147 - Added retained audit history for critical-event queue transitions

- Date: 2026-04-27
- Plan item: `epic-39 / effort-39.2`
- Status: completed

## Goal
Persist transition history for critical-event queue candidates so operators can inspect when candidates entered the queue and moved between monitoring, promoted, official-confirmation, and discarded states.

## Changes shipped
- Added `CRITICAL_EVENT_QUEUE_AUDIT_STATE_KEY` in `server.mjs`.
- Added critical-event queue audit helpers:
  - `getCriticalEventQueueAuditState()`
  - `saveCriticalEventQueueAuditState(state)`
  - `buildCriticalEventQueueAuditEntry(candidate, previous, nowIso)`
  - `appendCriticalEventQueueAuditEntries(entries, nowIso)`
- Updated `processCriticalEventQueue(snapshot)` so it now:
  - compares the previous retained candidate state with the current post-evaluation state
  - records audit entries when status, confidence, or promotion state changes
  - retains bounded transition history in memory-backed signal state
- Extended `buildCriticalEventQueueContract(snapshot)` so queue contracts now expose:
  - `audit.version = "critical-event-queue-audit-v1"`
  - audit `stateKey`
  - audit `updatedAt`
  - audit `totalEntries`
  - audit `recentTransitions`
- Added regression coverage proving queue transition audit entries are retained when candidate state changes occur.
- Marked `effort-39.2` complete and advanced local cycle tracking to 147.
