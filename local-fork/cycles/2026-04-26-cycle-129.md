# Crucix Dev Cycle 129 - Settings state bundle export/import and admin audit history

- Date: 2026-04-26
- Plan item: `epic-33 / effort-33.4`
- Status: completed

## Goal
Support config and state export-import plus operator audit history so deployments can be backed up, restored, or reviewed without manual file archaeology.

## Changes shipped
- Updated `server.mjs`
  - added settings admin audit storage at `runs/settings-admin-audit.json`
  - added helpers for:
    - settings audit recording and snapshots
    - settings payload summaries
    - deep-cloned state bundle export
    - settings-only or full bundle import
  - added bundle export mode at:
    - `GET /api/settings/export?mode=bundle`
  - added settings admin audit endpoint:
    - `GET /api/settings/audit`
  - extended admin settings surface with:
    - `stateBundle` capability
    - `auditHistory` capability
    - `latestAudit`
    - audit endpoint metadata
    - backup mode metadata
    - recent audit trail
  - audit-recorded actions now include:
    - settings writes
    - bundle exports
    - imports
  - bundle state currently includes:
    - operator settings
    - review acknowledgements
    - review workflow audit
    - cluster repair actions
    - noise suppression history
    - operational alerts state
- Updated `dashboard/public/admin-settings.html`
  - renamed plain export button to `Export settings`
  - added `Export bundle`
  - added `Import bundle`
  - surfaced latest audit/action metadata and recent admin audit entries
- Updated tests:
  - `test/operator-settings-persistence.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
