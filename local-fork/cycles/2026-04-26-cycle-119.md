# Crucix Dev Cycle 119 - Noise-suppression history decay and retention

- Date: 2026-04-26
- Plan item: `epic-29 / effort-29.5`
- Status: completed

## Goal
Add bounded retention or time-decay for noise-suppression history so one-off duplicate bursts and retired sources do not accumulate indefinite suggestion weight or unbounded state growth across repeated sweeps.

## Changes shipped
- Updated `server.mjs`:
  - added retention and half-life settings for persisted noise-suppression history
  - added bounded caps for retained duplicate-burst, low-value, and source-rule history buckets
  - prunes expired entries on load and on record
  - upgraded persisted history envelope to `noise-suppression-history-v2` with:
    - `retentionMs`
    - `halfLifeMs`
  - added decayed weighting for historical suggestion strength:
    - `decayedHistoryHitCount` on duplicate-burst candidates
    - `decayedHistoryHitCount` on repetitive-low-value candidates
    - `decayedHitCount` for suggested source rules
  - switched suggested source-rule gating from raw cumulative hits to decayed hit weight so stale sources stop accumulating indefinite influence
- Updated `test/cluster-repair-workflow.test.mjs`:
  - extended the harness for retention/decay constants and the v2 history envelope
  - added regression coverage for stale-history decay lowering suggestion weight
  - added regression coverage for expired history entries being pruned
- Updated `local-fork/BACKLOG.md`:
  - added a follow-on recommendation to surface decay/prune telemetry in `/api/health` or review diagnostics UI
