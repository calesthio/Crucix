# Crucix dev cycle 078

- Date: 2026-04-25
- Plan item: Epic 25, effort 25.3
- Title: Record pruning recommendations for noisy, redundant, expensive, or maintenance-heavy sources without auto-removing active production sources

## What changed

- Added a bounded pruning rubric at `source-ops/pruning-rubric.json` with five weighted dimensions:
  - noise pressure
  - duplication burden
  - maintenance cost
  - operator value retention
  - safe actionability
- Added a structured pruning assessment schema at `source-ops/schemas/pruning-assessment.schema.json`.
- Added an example pruning assessment at `source-ops/results/pruning/example-pruning-social-001.json` showing:
  - a recommendation-only review of active source `reddit`
  - cohort comparison against `Bluesky`, `Telegram`, and `GDELT`
  - explicit production guardrails blocking auto-removal and direct production mutation
- Extended `source-ops/profile.json` with canonical pruning paths:
  - `pruningSchemaPath`
  - `pruningRubricPath`
- Extended the pruning task packet so pruning work now points explicitly to the structured schema and rubric.
- Extended `source-ops/schemas/task-packet.schema.json` with a `pruning` contract block.
- Extended `lib/source-scorecard.mjs` with pruning rubric and pruning assessment summary helpers.
- Extended `lib/source-ops-runtime.mjs` so the operator-facing `sourceOps` surface now includes:
  - `pruning`
  - `examplePruning`
- Expanded validation coverage in:
  - `test/source-ops-contract.test.mjs`
  - `test/source-ops-runtime.test.mjs`
