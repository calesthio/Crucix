# Crucix Dev Cycle 099 - Operator source management console

- Date: 2026-04-26
- Plan item: `epic-30 / effort-30.1`
- Status: completed

## Goal
Build an operator-facing source management console for source enable-disable controls, category filtering, role grouping, and per-source selection without requiring raw file edits.

## Changes shipped
- Extended the operator settings contract with a dedicated source console section:
  - `sourceConsole.version = "source-console-v1"`
  - `sourceConsole.surface = "/source-ops"`
  - role grouping metadata sourced from fusion-role runtime summaries
  - category and lifecycle filter options
  - persisted selection state and source inventory items
- Extended source inventory items with source-console-friendly metadata:
  - `fusionRole`
  - `trustProfile`
- Added operator access metadata:
  - `access.sourceConsoleSurface = "/source-ops"`
- Added a new operator-facing HTML surface:
  - `dashboard/public/source-ops.html`
- Added a new route:
  - `GET /source-ops`
- Added source-ops navigation links from:
  - operator settings page
  - admin settings page
- The new source console provides:
  - category filter
  - role filter
  - lifecycle filter
  - name/ID search
  - visible enabled-state summary based on persisted source/category selection
  - per-source table with category, role, lifecycle, live state, trust profile, and source ID
