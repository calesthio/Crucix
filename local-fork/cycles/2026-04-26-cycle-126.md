# Crucix Dev Cycle 126 - Noise-suppression sustained-pressure alerting

## Goal
Add a sustained-pressure alert or queue threshold for noise-suppression telemetry so repeated retained-entry growth or consecutive prune-active sweeps escalate from diagnostics-only visibility into an operator action cue.

    - `minRetainedEntries`
    - `minRetainedDelta`
    - `minConsecutiveGrowthSweeps`
    - `minConsecutivePruneSweeps`

    - `escalationAfter`
  - added `summarizeNoiseSuppressionPressure()`

- Updated `dashboard/public/diagnostics.html`
