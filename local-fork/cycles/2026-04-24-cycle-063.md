# Crucix dev cycle 063

- Date: 2026-04-24
- Plan item: Epic 19, effort 19.2
- Title: Prioritize review items by region pressure, near-duplicate suspicion, and repeated low-confidence patterns so the queue is triage-friendly

## What changed

- Extended review decoration so review items carry both persistent review history and persistent pressure data by region.
- Upgraded `buildOperatorReviewQueue(review, { maxItems, quality })` to compute transparent triage priority scores using:
  - region pressure score
  - near-duplicate suspicion score
  - repeated split/low-confidence regional pattern counts
  - consecutive failures
  - severity and item-count weighting
  - chronic-failure bonus
- Queue items now expose:
  - `priorityScore`
  - `priorityDrivers`
  - `pressureScore`
  - `duplicateScore`
  - `splitCount`
- Updated `/api/data` and `/api/brief/news/review` queue generation to pass cluster quality review metrics into the prioritizer.
- Updated the dashboard `Review Queue` panel so each item shows operator-visible priority score and driver text instead of only the raw reason.
- Restarted the local Crucix runtime so the live service serves the prioritized queue payload and dashboard panel.
