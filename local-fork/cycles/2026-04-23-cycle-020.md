- Added a new unscheduled recommendation to `local-fork/BACKLOG.md`:

- Implemented the first Epic 5 slice in `dashboard/inject.mjs`:
  - added `extractBalancedJsonFragment()` to recover the first balanced JSON object or array from prose-wrapped model output
  - added `parseClusterResponse()` to accept either a top-level array or `{ items: [...] }`
  - added `attemptClusterRepair()` so malformed cluster output gets one repair pass before heuristic fallback
  - tightened the primary cluster prompt to request an explicit `{ "items": [...] }` object with no prose or markdown
  - added repair telemetry fields: `repairAttemptCount` and `repairSuccessCount`
- Exposed the new hardening metadata in `server.mjs`:
  - news cluster summary now includes repair counters
  - compact briefing text now reports repair count when non-zero
- Fixed a startup regression introduced by the heavier repair path:
  - startup snapshot loading now synthesizes with `newsLlmMode: off` so the runtime becomes available immediately
  - fresh sweeps still run the full LLM clustering path

## Evidence

- Synthetic helper validation confirmed:
  - `extractBalancedJsonFragment()` recovered a fenced `{ "items": [...] }` payload from prose-wrapped text
  - `parseClusterResponse()` parsed that payload successfully
- Live runtime validation confirmed startup availability after restart:
  - `/api/health` returned `status: ok`
  - `/api/brief/news` responded immediately from the startup snapshot instead of sitting at `503`
  - returned LLM summary showed `requestedMode: off`, `fallbackReason: operator-disabled`, and zero candidate/fallback/repair counts for the fast startup snapshot path
  - `/api/brief/compact` included `News LLM: mode off, heuristic fallback, candidates 0, fallbacks 0`
- Additional validation notes:
  - a forced live recomputation with the full LLM path did not finish within the direct command timeout window, so the repair path was verified synthetically rather than claimed as live-rescued on this cycle
  - this means the code path is shipped and startup-safe, but the next cycle should focus on reducing force-path latency and measuring actual repair-hit rate on live candidate sets
