# Crucix dev cycle 060

- Date: 2026-04-24
- Plan item: Epic 18, effort 18.2
- Title: Expose a compact runtime explanation surface for why deterministic fallback is active and whether a configured LLM path actually participated

## What changed

- Added shared server-side helper `buildRuntimeLlmStatus(snapshot, options)` to normalize runtime LLM operator state.
- The runtime LLM surface now exposes:
  - overall status and label
  - provider/model/configured state
  - compact summary of whether published output is applied, pending, fallback, unavailable, or merely available
  - per-surface breakdown for analysis and leverageable ideas, including whether each path attempted or actually participated
- Added `runtimeLlm` to:
  - `/api/data`
  - `/api/brief/compact`
  - `/api/health`
- Updated the dashboard right-rail Analysis Review panel to consume the new shared runtime surface and show:
  - normalized badge from `runtimeLlm.analysis.label`
  - compact explanation from `runtimeLlm.analysis.explanation`
  - overall runtime summary from `runtimeLlm.label` and `runtimeLlm.summary`
- Restarted the local Crucix runtime so the live service on `127.0.0.1:3117` serves the updated API and dashboard.
