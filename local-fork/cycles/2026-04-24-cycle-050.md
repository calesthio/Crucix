# Crucix Cycle 050 - 2026-04-24

## Summary
- Completed Epic 15 effort 15.1: fix sweep lifecycle honesty so operator freshness reflects live runtime state instead of stale capture-time state.
- The bug had two parts:
  - `agentAnalysis.freshness.sweepInProgress` could stay frozen from the moment the snapshot was built.
  - `sweepStartedAt` was not being cleared on sweep exit, which made health reporting less trustworthy.
- The production-safe fix was to sync served agent-analysis freshness fields to runtime truth and clear sweep-start state in the sweep finalizer.

## What changed
- Added `syncSnapshotRuntimeFreshness()` in `server.mjs`.
- `ensureCurrentData()` now normalizes stored `agentAnalysis.freshness` against live runtime state before responses are served.
- The sweep finalizer now:
  - sets `sweepInProgress = false`
  - clears `sweepStartedAt`
  - re-syncs the currently published snapshot freshness flags
- Added regression coverage in `test/sweep-lifecycle-honesty.test.mjs` for both:
  - stale captured `sweepInProgress: true` being corrected to live `false`
  - active runtime sweep state overriding a stale stored `false`
