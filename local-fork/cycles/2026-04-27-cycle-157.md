# Crucix Dev Cycle 157 - Tuned dashboard layout efficiency after live gap analysis

- Date: 2026-04-27
- Plan item: `epic-48 / effort-48.1`
- Status: completed

## Goal
Take the next dashboard backlog slice beyond map-only sizing by analyzing the live layout for actual space waste, narrow stacking, and awkward gaps, then tune desktop column balance and panel placement so the dashboard uses screen real estate more efficiently without breaking responsive degradation.

## Live analysis before changes
Using the running dashboard at `http://127.0.0.1:3117/` with a `1280x720` browser viewport, the biggest inefficiencies were measurable:

- Topbar height was about `154px`, consuming too much first-screen budget.
- Grid columns were `240 / 656 / 340`, which left the right rail too narrow for several operator panels while still giving the left rail more width than it truly needed.
- Lower center panels were arranged by flex sizing into awkward widths:
  - `newsTicker`: about `255px`
  - `macroMarkets`: about `391px`
  - `agentAnalysis`: full-width `656px`
  - `tradeIdeas`: full-width `656px`
- The right rail stacked many panels at only `340px` width, which made several sections feel cramped while the center column still accumulated a very tall scroll budget.
- In practice, this created the exact failure mode Jason called out: some sections were too narrow, some sections were overly tall, and the page spent height inefficiently even where pure horizontal gaps were not large.

## Changes shipped
Updated `dashboard/public/jarvis.html` to make a first desktop-efficiency pass:

### Desktop column rebalance
- Changed desktop grid columns from `240 / 1fr / 340` to `220 / 1fr / 380`.
- Changed the sub-1400 desktop grid from `240 / 1fr / 320` to `220 / 1fr / 300`.
- Result: less wasted width on the left rail and more usable width on the right rail where stacked diagnostic/operator panels were cramped.

### Topbar density tuning
- Reworked desktop topbar layout into a compact two-row grid:
  - row 1: left identity/status + right controls
  - row 2: center preset/region controls
- Reduced desktop topbar padding, control gaps, and control/button font padding.
- This brought the measured topbar height down substantially.

### Lower-panel placement tuning
- Replaced the desktop lower-center flex behavior with a deliberate two-column grid.
- Assigned panel placement intentionally:
  - `macroMarkets` spans full width first
  - `agentAnalysis` and `tradeIdeas` sit side by side
  - `newsTicker` spans full width below them
- Reordered the lower-panel render order to match that intended placement.
- This removed the previous awkward narrow ticker + oversized macro pairing and reduced the feeling of random stacking.

### Height control for long-scroll panels
- Added bounded desktop heights with internal scroll for the most vertically greedy panels:
  - `tradeIdeas`
  - `reviewQueue`
  - `evidenceAudit`
  - `rightOsint`
- This prevents a few long panels from becoming giant page-height drivers at the expense of the rest of the dashboard.

## Live validation after changes
Revalidated the running dashboard at `http://127.0.0.1:3117/` with the same `1280x720` viewport.

Measured improvements:
- Topbar height dropped from about `154px` to about `107px`.
- Grid columns became `220 / 636 / 380`.
- Map stayed bounded and healthy at about `636x420`.
- Lower-center panel widths became more intentional:
  - `macroMarkets`: `636px`
  - `agentAnalysis`: `313px`
  - `tradeIdeas`: `313px` with bounded height around `403px`
  - `newsTicker`: `636px`
- Right rail panels gained usable width and bounded heights:
  - `reviewQueue`: `380px` wide, bounded around `403px`
  - `evidenceAudit`: `380px` wide, bounded around `403px`
- Total grid height dropped materially versus the pre-change pass because long-scroll panels stopped dominating the page.
