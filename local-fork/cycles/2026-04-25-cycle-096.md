# Crucix Dev Cycle 096 - Stuck sweep classification and recovery telemetry

- Date: 2026-04-25
- Plan item: `epic-29 / effort-29.2`
- Status: completed

## Goal
Detect, classify, and recover stuck sweep or synthesis jobs so `/api/health` exposes the live phase and watchdog recovery can distinguish briefing, synthesis, and deferred analysis hangs instead of leaving operators with a generic stuck `sweepInProgress` gate.

## Changes shipped
- Added runtime phase state tracking for long-running sweep work:
  - current phase
  - phase start time
  - last completed phase/time
  - last failure phase/time/reason
  - last recovery phase/time/reason
- Extended `getSweepWatchdogSnapshot()` to expose:
  - `phase`
  - `phaseStartedAt`
  - `lastCompletedPhase`
  - `lastCompletedAt`
  - `lastFailurePhase`
  - `lastFailureAt`
  - `lastFailureReason`
  - `lastRecoveryPhase`
  - `lastRecoveryAt`
  - `lastRecoveryReason`
  - `recoveryClassification`
- Watchdog recovery now classifies overdue jobs instead of using a generic reason string, including:
  - `source-sweep-hang`
  - `synthesis-hang`
  - `deferred-analysis-hang`
  - fallback `sweep-gate-hang`
- Recovery broadcasts now include `recoveredPhase`.
- Sweep lifecycle now marks phase transitions through:
  - `briefing`
  - `synthesis`
  - deferred `llm-analysis-refinement`
- Runtime failures now retain the failing phase and reason for operator inspection.
