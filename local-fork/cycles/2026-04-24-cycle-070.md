# Crucix dev cycle 070

- Date: 2026-04-24
- Plan item: Epic 23, effort 23.1
- Title: Define a canonical source registry schema that supports active, candidate, shadow, rejected, and deprecated lifecycle states plus agent-review provenance

## What changed

- Added a canonical source registry schema at `source-ops/source-registry.schema.json`.
- Added a registry builder at `lib/source-registry.mjs` that materializes a `source-registry-v1` document from current Crucix source policy.
- Added a seeded registry artifact at `source-ops/source-registry.seed.json` generated from the current runtime source set.
- The schema now defines a first-class record contract for each source, including:
  - canonical source id
  - runtime name
  - module path
  - category
  - trust class
  - evidence mode
  - freshness target
  - lifecycle state
  - operator role
  - enabled-by-default status
  - review provenance and review status
- Added operator-role inference so current runtime sources are seeded into meaningful initial roles such as `anchor`, `corroborator`, `anomaly-detector`, `context`, and `exploratory`.
