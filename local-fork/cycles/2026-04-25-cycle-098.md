# Crucix Dev Cycle 098 - Deferred job timeout and completion telemetry

- Date: 2026-04-25
- Plan item: `epic-29 / effort-29.4`
- Status: completed

## Goal
Add timeout, retry, cancellation, and completion telemetry for synthesis and deferred LLM work so slow, hung, and failed jobs are operationally distinguishable instead of showing only coarse sweep state.

## Changes shipped
- Added bounded runtime job telemetry buckets for:
  - `synthesis`
  - `ideas`
  - `analysis`
- Each job now tracks:
  - `active`
  - `attemptCount`
  - `retryCount`
  - `cancellationCount`
  - `lastAttemptId`
  - `lastStartedAt`
  - `lastCompletedAt`
  - `lastDurationMs`
  - `lastOutcome`
  - `lastError`
  - `lastTimedOut`
  - `timeoutMs`
- Added helper functions for runtime job lifecycle:
  - `beginRuntimeJob()`
  - `completeRuntimeJob()`
  - `getRuntimeJobsSnapshot()`
- Wired telemetry into live execution paths:
  - sweep/synthesis lifecycle in `runSweepCycle()`
  - deferred ideas generation in `enrichIdeasAndPublish()`
  - deferred agent-analysis refinement in `enrichAgentAnalysisAndPublish()`
- Extended `runtime-control-v1` so both `/api/health` and admin settings expose `runtimeControl.jobs`.
- Preserved test-safety by making runtime-control contract builders degrade cleanly when extracted into VM harnesses without full server globals.
