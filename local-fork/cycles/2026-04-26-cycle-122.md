# Crucix Dev Cycle 122 - Runtime-bucket drift surfacing

- Date: 2026-04-26
- Plan item: `epic-32 / effort-32.8`
- Status: completed

## Goal
Surface source-registry runtime-bucket metadata and registry-vs-observed attribution drift in the source-ops console so operators can review when a source declared single-publisher starts behaving like an aggregator or when new aggregator aliases should be added to the contract.

## Changes shipped
- Updated `lib/source-ops-runtime.mjs`:
  - added shared runtime-bucket drift diagnostics under `performance.workflow.attributionDiagnostics.runtimeBucketDrift`
  - detects `single-publisher-mismatch` when a declared single-publisher source is observed with multiple publisher labels
  - detects `missing-aggregator-alias` when an expected multi-publisher source observes publisher labels outside its declared alias contract
  - added summary counters for total drift, single-publisher mismatches, and missing aggregator aliases
- Updated `server.mjs`:
  - wired runtime-bucket drift into the source-console payload
  - attached per-source runtime-bucket metadata and any matched drift item to source-console inventory rows
- Updated `dashboard/public/source-ops.html`:
  - added a `Runtime bucket contract and drift` section
  - surfaced runtime-bucket drift counts in the performance summary stats
  - added `Runtime bucket` and `Observed drift` columns to the source table
- Updated tests:
  - `test/source-ops-runtime.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
- Updated `local-fork/BACKLOG.md`:
  - added a follow-on recommendation for persisting runtime-bucket drift history and delta trends across sweeps
