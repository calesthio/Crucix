# Crucix Cycle 043 - 2026-04-24

## Summary
- Completed Epic 11 effort 11.1: persist rolling per-region prompt-tuning and retry pressure stats across restarts.
- Added a persistent pressure-state layer for news LLM clustering so operators can see which regions historically needed retries, backoff, repair attempts, or custom tuning even after restart.
- Exposed the pressure summary on the review stats and health surfaces, and decorated live per-region entries with matching persistent pressure context.

## What changed
- Added persistent pressure state in `server.mjs`:
  - `getClusterPressureStatsState()`
  - `pruneClusterPressureRegions()`
  - `summarizeClusterPressureStats()`
  - `recordClusterPressureStats()`
  - `attachClusterPressureStats()`
- Pressure state tracks per-region:
  - retry count
  - backoff count
  - repair attempt count
  - tuning count
  - heuristic fallback count
  - success count
  - max configured retries
  - max repair timeout
  - last/current tuning metadata
- Extended surfaces:
  - `buildNewsClusterSummary(...).llm.persistentPressure`
  - `buildNewsClusterSummary(...).llm.perRegion[*].persistent`
  - `GET /api/brief/news/review/stats` now includes `pressure`
  - `GET /api/health` now includes `clusterPressureStats`
- During synthesis/run completion, pressure state is recorded and attached to `synthesized.newsLlmDebug`.
- Added regression coverage in `test/news-cluster-pressure-persistence.test.mjs`.
