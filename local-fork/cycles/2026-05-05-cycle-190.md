# Crucix Dev Cycle 190 - Lazy-load 3D dashboard dependencies to reduce browser memory pressure

- Date: 2026-05-05
- Plan item: `epic-71 / effort-71.1`
- Status: completed locally

## Goal
Stop the dashboard from paying the memory cost of `three.js` and `globe.gl` during ordinary flat-mode loads, with Safari treated conservatively so the live site is less likely to be reloaded under browser memory pressure.

## Backlog review and promotions
- Added a new direct roadmap slice as `epic-71 / effort-71.1` for dashboard 3D dependency lazy loading and Safari memory-pressure mitigation.
- No separate backlog promotion was needed because this work was requested directly from the live issue report.

## Findings
- A real front-end memory problem existed in the shipped dashboard path: `dashboard/public/jarvis.html` eagerly loaded both `three.js` and `globe.gl` even when the session started and stayed in flat mode.
- The dashboard already had a flat-map-first posture on many layouts, so paying the 3D dependency cost up front was unnecessary memory pressure.
- Safari is the browser most likely to surface the observed operator-facing symptom, `This webpage was reloaded because it was using significant memory`, so a conservative default there is warranted.

## Changes shipped
- Removed eager top-level script tags for `three.js` and `globe.gl` from `dashboard/public/jarvis.html`.
- Added `loadExternalScript()` and `ensure3DLibs()` so the 3D stack is loaded only when globe mode is actually needed.
- Updated `initMap()` to keep true flat mode flat, instead of treating missing 3D libraries as an automatic software-globe path during initial load.
- Updated `toggleMapMode()` so moving into globe mode loads the 3D stack on demand, while preserving the existing software-globe fallback if WebGL or script loading fails.
- Added Safari detection and defaulted Safari sessions to lite visuals by default, reducing animation and rendering pressure without lying about capability.
- Updated local roadmap state in `local-fork/plan.json` and recorded this cycle as sequence `190`.

- Add a targeted dashboard regression test that asserts eager 3D script tags are absent from the served page while lazy-loader helpers remain present.
