# Crucix Dev Cycle 143 - Added proactive critical-event routing contract

- Date: 2026-04-27
- Plan item: `epic-35 / effort-35.4`
- Status: completed

## Goal
Expose proactive routing for vetted critical-event candidates so operator-facing contracts show who would be notified, over which routes, with what confidence, corroboration basis, and unresolved caveats.

## Changes shipped
- Added `critical-event-routing-v1`.
- Exposed routing on:
  - `GET /api/settings` under `alerts.criticalEvents.routing`
  - `GET /api/health` as `criticalEventRouting`
  - `GET /api/data` as `criticalEventRouting`
- Routing contract now exposes:
  - `activeRoutes`
  - `defaultRoute`
  - `escalationRoute`
  - `eligibleCount`
  - `eligibleCandidates`
  - `dispatchMode`
  - per-candidate `messagePreview`
  - per-candidate `corroborationBasis`
  - per-candidate `unresolvedCaveats`
- Kept this slice in `dispatchMode = "contract-only"` so the system exposes vetted routing decisions without silently emitting outbound notifications during validation.
- Updated contract tests to cover routing exposure across operator settings, health, and UI-facing payloads.
- Updated `local-fork/plan.json` to mark `effort-35.4` complete and advance cycle tracking to 143.
