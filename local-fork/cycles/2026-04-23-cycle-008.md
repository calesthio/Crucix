## Results

- Added a new backlog item for stable signal IDs in compact/drill-down responses so future chat clients do not have to rely on list order.
- Added `getSignalSelection()` and `buildIMessengerDrilldown()` in `server.mjs`.
- Added a new runtime endpoint:
  - `/api/brief/drilldown`
- Supported drill-down actions:
  - `why`
  - `sources`
  - `expand`
- Supported kinds:
  - `corroborated`
  - `suspect`
- The endpoint returns both compact human-readable text and the backing structured item.
