# Crucix dev cycle 064

- Title: Surface review queue state clearly in dashboard and API responses, including empty-queue semantics that explain why metrics may still be elevated

## What changed

- Extended `buildOperatorReviewQueue(review, { ... })` to publish explicit queue-state semantics.
- Queue payloads now expose:
  - `state`
  - `hasElevatedMetrics`
  - `emptyReason`
  - summarized queue `metrics`
- Added explicit states for:
  - `active`
  - `empty_clear`
  - `empty_elevated_metrics`
- Empty queue cases now distinguish between:
  - nothing actionable and no notable review pressure
  - nothing actionable right now, but background review metrics remain elevated
- Updated the dashboard `Review Queue` panel to render the elevated-empty explanation and show compact metric counts when the queue is empty but review pressure is still present.
- Restarted the local Crucix runtime so the live service serves the updated queue-state semantics.
