# Crucix Dev Cycle 107 - LLM cost, latency, and completion telemetry

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.4`
- Status: completed

## Goal
Add operator-visible cost, latency, and completion telemetry for LLM-backed clustering and analysis so expensive or degraded paths can be diagnosed quickly.

## Changes shipped
- Added shared LLM telemetry helpers in `lib/llm/telemetry.mjs` for:
  - usage normalization
  - estimated cost calculation
  - per-call telemetry envelopes
  - aggregate telemetry summaries
- Extended clustering telemetry capture in `dashboard/inject.mjs` for:
  - clustering generation calls
  - clustering repair calls
  - aggregate latency, token, cost, and completion summaries
- Extended agent-analysis LLM metadata in `server.mjs` to capture:
  - per-call latency
  - completion outcome
  - usage and cost summary
  - unavailable or failed-call telemetry states
- Extended `/api/llm/operations` with operator-visible `llmTelemetry` for:
  - clustering aggregate
  - clustering generation
  - clustering repair
  - agent analysis
- Extended the clustering debug contract to expose the same telemetry summary for debugging correlation.
- Updated `dashboard/public/llm-ops.html` to render an `LLM telemetry` card with operator-visible cost, latency, and completion details.
- Expanded regression coverage in:
  - `test/settings-contract.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
