# Crucix dev cycle 091 - 2026-04-25

## Plan target
- Epic 28, effort 28.1
- Replace the current brittle panel placement with a responsive layout system that properly fits the newer content windows and supports narrow, desktop, and wallboard display modes.

## What changed
- Added a first-class persisted dashboard layout display mode under operator settings:
  - `layout.displayMode`
  - allowed values: `auto`, `narrow`, `desktop`, `wallboard`
- Extended operator/admin settings contracts so layout controls now expose:
  - `displayMode`
  - `availableDisplayModes`
- Updated the admin settings surface so operators can persist a display mode intentionally instead of relying only on viewport breakpoints.
- Reworked `jarvis.html` layout handling so the dashboard now resolves an explicit runtime layout mode and applies it via `body[data-layout-mode]`.
- Added real responsive layout profiles:
  - `narrow` for stacked single-column operation
  - `desktop` for the standard multi-column operator view
  - `wallboard` for denser large-screen use with expanded map height and a multi-column lower grid
- Added a topbar layout badge so the active layout mode is visible during runtime.
- Kept `auto` mode so layout still follows viewport width when no explicit operator preference is set.
