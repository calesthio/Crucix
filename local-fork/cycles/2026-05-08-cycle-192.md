# Crucix Dev Cycle 192 - X acquisition ladder and graceful degradation

- Date: 2026-05-08
- Plan item: `epic-72 / effort-72.2`

## Goal
Implement the first bounded X acquisition ladder slice so Crucix can record requested retrieval tiers, degrade gracefully when a tier is unavailable, and preserve explicit manual evidence gaps instead of silently pretending retrieval succeeded.

## Backlog review and promotions
- Added a follow-up backlog recommendation to ship a social-leads policy artifact and endpoint before browser-assisted X retrieval is enabled.

- The smallest safe 72.2 slice is contract-level acquisition routing and graceful degradation, not actual unattended retrieval.
- URL-only X leads are still useful if Crucix records them honestly as manual-evidence gaps with explicit next actions.

## Changes shipped
