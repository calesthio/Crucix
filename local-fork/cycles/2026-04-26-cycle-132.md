# Crucix Dev Cycle 132 - Added SQLite runtime-history diagnostics surface

- Date: 2026-04-26
- Plan item: `epic-33 / effort-33.7`
- Status: completed

## Goal
Add a small admin diagnostics surface for the SQLite runtime-history store so datastore health is visible without shell access.

## Changes shipped
- Extended `lib/delta/runtime-history-store.mjs` with `getDiagnostics()` to expose:
  - bounded `quick_check(1)` integrity status
  - SQLite page and file size metadata
  - per-table row counts and first/last write timestamps
  - bounded recent samples for `runtime_runs` and `runtime_signal_state`
- Added `GET /api/runtime-history/diagnostics` in `server.mjs` behind `requireDebugAccess`.
- Extended the admin settings contract so local admin tooling can discover the new diagnostics endpoint.
- Added regression coverage for the store diagnostics and live endpoint/admin contract exposure.
- Updated `local-fork/plan.json` to mark `effort-33.7` complete and close out `epic-33`.
