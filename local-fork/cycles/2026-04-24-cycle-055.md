# Crucix Cycle 055 - 2026-04-24

## Summary
- Completed Epic 16 effort 16.3 by hardening dashboard fallback messaging for failed visual layers, specifically the globe-to-software-globe fallback path.
- The degraded state is now explicit, bounded, and operator-readable instead of being compressed into a small ambiguous hint string.

## What changed
- Updated `dashboard/public/jarvis.html`.
- Added a bounded map status card:
  - `#mapStatus`
  - `#mapStatusKicker`
  - `#mapStatusText`
  - `#mapStatusMeta`
- Updated `updateMapModeControls()` so when software globe fallback is active it now:
  - keeps the primary control as `FLAT MODE`
  - uses a short hint (`SOFTWARE GLOBE ACTIVE`)
  - shows a bounded status card with:
    - fallback state label
    - concise operator explanation
    - explicit failure cause / mode metadata
- This keeps the fallback message visible and honest without polluting the main map control or looking like a broken rendering artifact.
