# Crucix Dev Cycle 145 - Made restart-safe prove live PID rotation honestly

- Date: 2026-04-27
- Plan item: `epic-37 / effort-37.1`
- Status: completed

## Goal
Make local `restart-safe` stop lying about deploy outcome by proving that port 3117 rotated to a new healthy listener PID, or failing explicitly when the original listener never relinquished ownership.

## Changes shipped
- Added `evaluateRestartTransition(previousListeners, nextListeners, options)` to `lib/runtime-restart.mjs`.
- Extended restart transition logic to classify:
  - `cleared`
  - `replacement-detected`
  - `waiting`
  - `foreign-listener`
- Updated `scripts/restart-helper.mjs` so it:
  - tracks the original listener PID set
  - distinguishes a true stuck shutdown from a successful replacement listener appearing during shutdown
  - refuses foreign listeners that take the port during restart
  - only reports success when health passes and PID rotation is actually proved
  - reports `replacementMode` and `rotationProved` in the final JSON result
- Promoted remaining backlog items into future roadmap epics:
  - Epic 39 for critical-event classifier hardening, transition audit, and delivery audit
  - Epic 40 for SDR session automation and retained RF evidence
- Added new backlog item `backlog-2026-04-27-runtime-control-restart-audit` for operator-visible retained restart outcome history.
- Updated `local-fork/plan.json` to mark `effort-37.1` complete and advance cycle tracking to 145.

## Validation
- Tests passed:
  - `node --test test/runtime-restart-helper.test.mjs test/settings-endpoint-contract.test.mjs`
- Live restart validation passed:
  - `npm run restart:safe`
  - helper output showed:
    - `Stopping existing Crucix listener PID 3386 on port 3117.`
    - `Detected replacement listener PID 14916 while original PID(s) were shutting down.`
    - `Health check passed for http://127.0.0.1:3117/api/health.`
    - final JSON included `"livePid": 14916`, `"replacementMode": "replacement-detected"`, and `"rotationProved": true`

## Backlog and follow-up

## Notes
- The previous failure mode was real: a replacement listener could appear during shutdown, causing the old helper to report failure even though the port had rotated to a new healthy Crucix PID.
- This slice fixes the honesty problem without pretending restart orchestration is more deterministic than it is. The contract now treats replacement during shutdown as success only when the live PID is different and healthy.
