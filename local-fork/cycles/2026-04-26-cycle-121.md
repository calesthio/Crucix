# Crucix Dev Cycle 121 - Noise-suppression telemetry trends

- Date: 2026-04-26
- Plan item: `epic-29 / effort-29.7`
- Status: completed

## Goal
Add rolling per-sweep noise-suppression telemetry snapshots and a lightweight diagnostics trend view so operators can distinguish one-off aged-out or prune activity from sustained state-growth pressure over time.

## Changes shipped
- Updated `server.mjs`:
  - added `buildNoiseSuppressionTelemetrySnapshot(...)`
  - stores a compact per-sweep `noiseSuppressionTelemetrySnapshot` on each synthesized run
  - exposes `noiseSuppressionTrend` on `/api/health`
  - exposes `workflow.noiseSuppression.trend` on review workflow and source-console surfaces
- Updated `lib/delta/memory.mjs`:
  - persists compact noise-suppression telemetry snapshots in run memory
  - reconstructs trend history and delta views from retained hot/cold compact memory
- Updated `dashboard/public/diagnostics.html`:
  - added a lightweight trend section for recent noise-suppression sweeps
  - shows retained/total entries, aged-out counts, prune activity, candidate counts, and delta-vs-previous-sweep rows
- Updated tests:
  - `test/noise-suppression-history-trend.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
  - `test/sweep-lifecycle-transitions.test.mjs`
- Updated `local-fork/BACKLOG.md`:
  - added a follow-on recommendation for a simple sustained-pressure alert or queue threshold when repeated retained-entry growth or consecutive prune-active sweeps persist
