# Crucix Dev Cycle 112 - Cluster repair helpers for weak clusters

- Date: 2026-04-26
- Plan item: `epic-32 / effort-32.2`
- Status: completed

## Goal
Add cluster repair helpers that explain why a cluster is weak and support bounded merge, split, placement-correction, and suppression actions.

## Changes shipped
- Added persisted cluster-repair decision tracking in `server.mjs`:
  - `runs/cluster-repair-actions.json`
  - bounded decision history
  - suppressed-cluster tracking
- Added `cluster-repair-workflow-v1` construction in `server.mjs` with:
  - weak cluster detection
  - bounded weakness reasoning
  - merge candidate helpers from suspicious near-duplicate pairs
  - split recommendations for large heuristic clusters
  - placement-correction recommendations for weak placement precision
  - suppression actions for repetitive or low-value clusters
  - recent repair decisions
- Extended the review workflow contract so `/api/brief/news/review` now exposes:
  - `workflow.clusterRepair`
  - supported repair actions
  - weak-cluster counts and metrics
- Extended `POST /api/review-workflow/action` to support:
  - `merge-clusters`
  - `split-cluster`
  - `correct-placement`
  - `suppress-cluster`
- Updated `dashboard/public/diagnostics.html` to render a weak-cluster repair helpers card with direct bounded action buttons.
- Added regression coverage in `test/cluster-repair-workflow.test.mjs`.
