## Results

- Added a new backlog item for optional guardrails on debug endpoints, such as local-only enablement or a config flag, if the runtime is ever exposed beyond loopback.
- Added new selection-memory helpers in `server.mjs`:
  - `resetSelectionMemoryTelemetry()`
  - `selectionMemorySnapshot(limit)`
- Added new debug/reset endpoints:
  - `GET /api/brief/context/debug?limit=N`
  - `POST /api/brief/context/telemetry/reset`
- The debug endpoint returns:
  - current aggregate selection-memory stats
  - a bounded snapshot of recent contexts (default 5, capped at 20)
- The reset endpoint clears telemetry counters without deleting remembered context entries.

## Notes

- The debug endpoint is intentionally lightweight and bounded. It shows recent context state for operator inspection, not a full unbounded dump.
- No guard flag was added in this slice because the runtime is currently loopback-bound; backlog now tracks that hardening step if exposure changes later.
