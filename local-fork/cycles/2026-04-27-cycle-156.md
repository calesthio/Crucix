# Crucix Dev Cycle 156 - Added live dashboard layout-budget diagnostics

- Date: 2026-04-27
- Plan item: `epic-47 / effort-47.1`
- Status: completed

## Goal
Promote the next backlog issue after map tokenization and expose a measurable dashboard layout-budget surface so map sizing, topbar cost, and first-screen budget can be checked from live DOM state instead of only by eyeballing screenshots.

## Changes shipped
- Promoted dashboard layout-budget diagnostics into `local-fork/plan.json` as `epic-47 / effort-47.1` and completed it in the same cycle.
- Updated `dashboard/public/jarvis.html` to add a new `Layout Budget` operator panel in the right rail.
- Added live measurement helpers that read the current DOM and surface:
  - current layout mode
  - viewport width and height
  - measured map size
  - active map sizing tokens
  - topbar share of viewport
  - map share of viewport
  - combined first-screen share for topbar plus map
  - remaining viewport after topbar plus map
  - lower-grid height and center-column overflow for scroll context
- Added `layoutBudget` panel defaults to workspace presets so diagnostics mode pins it prominently and operator/source-ops modes can still surface it when useful.
- The panel is intentionally sourced from live DOM measurements, not static assumptions, so it acts as an honest contract surface for future layout tuning.
