# Crucix Dev Cycle 163 - Restored map surfacing for urgent Telegram incidents

- Date: 2026-04-28
- Plan item: `epic-36 / effort-36.6`
- Status: completed

## Goal
Investigate and harden the missing map-surfacing path for high-salience breaking open-source events, especially incidents that appear first in urgent Telegram/OSINT channels and were previously visible in the ticker but absent from clustered map placement.

## Findings
- The backlog review found no unpromoted candidate items. The only open backlog item remained browser regression coverage for admin preset authoring.
- The weak link was in `dashboard/inject.mjs`: `buildNewsClusters()` only consumed RSS output from `fetchAllNews()`, while urgent Telegram posts were kept separate for the ticker via `buildNewsFeed()` and were emitted with `region: 'OSINT'` only.
- Result: a geocodable urgent Telegram incident could be visible in the feed text but never enter clustering, never receive map placement, and never expose Telegram provenance in cluster review surfaces.

## Changes shipped
- Added `buildTelegramNewsCandidates(posts)` in `dashboard/inject.mjs` to convert geocodable urgent Telegram posts into news-like cluster candidates with:
  - preserved channel attribution
  - explicit `type: 'telegram'`
  - direct coordinates from keyword geocoding
  - `placementBasis: 'telegram-urgent'`
  - inherited placement class and precision for later review surfaces
- Updated synthesis so news clustering now uses:
  - `rssNews = await fetchAllNews()`
  - `news = [...tgUrgentNews, ...rssNews]`
- This keeps urgent OSINT incidents eligible for clustering and map placement without disturbing the existing ticker feed path.
- Added regression coverage proving:
  - urgent Telegram incidents remain eligible for mapped clustering
  - Telegram-backed clusters preserve `placementBasis: 'telegram-urgent'`
  - exact Telegram-derived coordinates are preferred over generic regional aliases such as `United States`
  - Telegram source provenance remains visible as `runtimeSource: 'Telegram'`
