# Crucix dev cycle 090 - 2026-04-25

## Plan target
- Epic 27, effort 27.4
- Add role-separated admin controls and local-safe access boundaries so sensitive settings, debug surfaces, and runtime controls are not mixed with read-only operator views.

## What changed
- Split the settings control plane into two explicit surfaces:
  - `/settings` for the read-only operator view
  - `/admin/settings` for local-only admin writes and export/import actions
- Added `buildAdminSettingsContract()` and `GET /api/settings/admin` behind `requireDebugAccess`.
- Changed the operator contract returned by `GET /api/settings` so it now reports:
  - `access.role = operator`
  - `access.mode = read-only`
  - persistence write/export/import capabilities disabled
  - admin surface pointers without exposing the operator settings file path
- Kept persisted-write controls, export, and import on the admin side only, with:
  - `access.role = admin`
  - `access.mode = local-write`
  - persistence path and write/export/import capabilities exposed only on the admin contract
- Reworked `dashboard/public/settings.html` into a read-only operator visibility plane.
- Added `dashboard/public/admin-settings.html` as the local-only admin control plane with Save and Export controls.
- Expanded contract and booted-server tests to verify operator/admin separation and local-safe boundaries.

## Validation
- Focused test suite passed:
  - `node --test test/settings-contract.test.mjs test/settings-endpoint-contract.test.mjs test/operator-settings-persistence.test.mjs test/runtime-config-contract.test.mjs`
- Live runtime validation on `http://127.0.0.1:3117` after listener refresh confirmed:
  - `GET /api/settings` returns `version = operator-settings-v1`, `role = operator`, `writeApi = false`, `export = false`
  - `GET /api/settings/admin` returns `version = admin-settings-v1`, `role = admin`, `writeApi = true`, `export = true`
  - `/settings` does not include Save or Export controls
  - `/admin/settings` includes Save and Export controls

## Notes
- The recurring port `3117` stale-listener handoff problem happened again during restart validation. The already-planned restart helper / port-ownership sanity-check work remains operationally justified.
- Added a new backlog recommendation for an explicit local admin nonce or CSRF-style confirmation token on browser-driven local write endpoints, since locality alone is a thin boundary.
