# Crucix Dev Cycle 095 - Dev-safe restart helper and listener ownership checks

- Date: 2026-04-25
- Plan item: `epic-29 / effort-29.1`
- Status: completed

## Goal
Add a dev-safe restart helper with port ownership sanity checks so local validation cycles stop binding to stale `3117` listeners and can prove which instance is live after restart.

## Changes shipped
- Added `lib/runtime-restart.mjs` with reusable listener parsing and ownership classification helpers.
- Added `scripts/restart-helper.mjs` for conservative local restarts that:
  - inspect the current `3117` listener
  - verify ownership against the Crucix repo and `server.mjs`
  - refuse to kill foreign listeners
  - stop the owned Crucix listener cleanly
  - start a fresh detached Crucix process
  - verify health on the restarted listener
  - report started and live PID identity
- Added npm script:
  - `npm run restart:safe`
- Extended `/api/health` with `runtimeIdentity` including:
  - `pid`
  - `port`
  - `cwd`
  - `startedAt`
- Added startup logging for runtime identity so listener provenance is visible in logs.

## Notes
- The first helper run correctly refused to kill a listener when `lsof` output alone could not prove ownership. I then hardened ownership detection by enriching listeners with `ps` command-line inspection before allowing restart.
- This keeps the helper conservative instead of turning it into a blind `kill whatever owns 3117` button.
