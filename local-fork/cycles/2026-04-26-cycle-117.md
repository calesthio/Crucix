# Crucix Dev Cycle 117 - Source-registry runtime bucket metadata

- Date: 2026-04-26
- Plan item: `epic-23 / effort-23.4`
- Status: completed

## Goal
Add explicit source-registry metadata for expected multi-publisher runtime buckets so attribution diagnostics can classify aggregators like `GDELT` from data contract rather than a hard-coded heuristic list.

## Changes shipped
- Updated `lib/source-registry.mjs`:
  - added explicit `runtimeBucket` metadata generation for canonical source registry entries
  - added `SOURCE_RUNTIME_BUCKET_KINDS`
  - seeded `GDELT` as `expected-multi-publisher` with aliases `GDELT`, `RSS`, and `news`
  - seeded single-publisher runtime-bucket metadata for the rest of the canonical registry
- Updated `source-ops/source-registry.schema.json`:
  - added schema validation for `runtimeBucket`
  - requires `kind`, `attributionAliases`, and `rationale`
- Updated `source-ops/source-registry.seed.json`:
  - carried explicit `runtimeBucket` metadata into the seed contract
- Updated `lib/source-ops-runtime.mjs`:
  - attribution diagnostics now classify expected multi-publisher buckets from registry metadata rather than a hard-coded token list
  - source-ops inventory and performance surfaces now carry `runtimeBucket`
- Extended tests:
  - `test/source-registry.test.mjs`
  - `test/source-ops-runtime.test.mjs`

## Validation
