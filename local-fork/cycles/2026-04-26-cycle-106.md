# Crucix Dev Cycle 106 - Source-performance history and delta views

- Date: 2026-04-26
- Plan item: `epic-30 / effort-30.5`
- Status: completed

## Goal
Add historical source-performance trend snapshots and delta views so operators can compare attribution, cluster quality, review pressure, trust-outcome shifts, and top source movement across sweeps instead of inspecting only the current snapshot.

## Changes shipped
- Extended compact trend memory in `lib/delta/memory.mjs` to persist a bounded `sourcePerformanceSnapshot` alongside existing health and review-pressure history.
- Added compatibility normalization for older runs so legacy compact history remains readable when source-performance snapshots are absent.
- Added `getSourcePerformanceHistory(limit = 8)` to reconstruct:
  - versioned snapshot history
  - per-sweep summary blocks
  - trust-outcome deltas
  - cluster-quality and review-pressure deltas
  - top source shift summaries between adjacent sweeps
- Extended `lib/source-ops-runtime.mjs` so the source-ops surface now exposes `performanceHistory` in a trimmed operator-ready contract.
- Updated `server.mjs` so synthesized snapshots capture source-performance workflow data before memory compaction and publish historical performance history after each sweep.
- Updated `dashboard/public/source-ops.html` to show:
  - historical snapshot count
  - latest delta view
  - trust-outcome, cluster-quality, and review-pressure deltas
  - recent snapshot summaries
  - top source shifts
- Expanded regression coverage in:
  - `test/source-ops-runtime.test.mjs`
  - `test/source-performance-history.test.mjs`
