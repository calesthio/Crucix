# Crucix dev cycle 014

- Date: 2026-04-23
- Objective: improve selection-memory eviction quality with access-aware LRU behavior, and update backlog as needed
- Status: completed

## Planned work

1. Add any new unscheduled recommendations to the backlog.
2. Add access-aware LRU behavior for selection-memory eviction.
3. Validate the runtime endpoints and commit the cycle.

## Results

- Added a new backlog item for lightweight eviction/prune counters so operator health views can distinguish TTL expiry from capacity pressure.
- Added access-aware selection-memory behavior in `server.mjs`:
  - `touchSelection()`
  - stored `lastAccessAt` per context
- Updated selection-memory behavior so reads refresh recency by moving a touched context to the back of the `Map`, making overflow eviction effectively LRU instead of simple insertion-order trimming.
- Extended metadata surfaces to expose access state:
  - `selectionMeta()` now returns `lastAccessAt`
  - `selectionMemoryStats()` now returns `oldestContext` and `newestContext`
- This keeps active chat contexts warmer under churn while preserving the existing bounded-memory cap and TTL behavior.

## Evidence

- Synthetic helper validation confirmed:
  - after inserting `a`, `b`, `c`, touching `a`, then adding `d` with max set to three, the surviving keys were `c`, `a`, `d`
  - `b` was evicted, showing that access recency changed eviction order
  - `selectionMeta('a')` exposed `lastAccessAt`
- Live runtime verification confirmed:
  - repeated access to `imessage-lru-a` updated `lastAccessAt`
  - `GET /api/brief/context/health` exposed `oldestContext` and `newestContext`
  - `GET /api/brief/context?context=...` exposed per-context `lastAccessAt`
- Active runtime check:
  - one `server.mjs` process verified with `ps`
  - `/api/health` returned `status: ok`

## Notes

- Context inspection itself now counts as access and refreshes recency because it resolves through the same in-memory selection path. That is acceptable for now and consistent with an operator actively examining a live context.
- If later needed, a read-only peek path that does not affect recency could be added as a separate slice.
