# Crucix Dev Cycle 160 - Added wallboard render-budget telemetry and bounded panel virtualization

- Date: 2026-04-28
- Plan item: `epic-36 / effort-36.3`
- Status: completed

## Goal
Add panel-level render budget telemetry and optional wallboard virtualization so long news, OSINT, evidence, and review streams stop silently dragging the dashboard as the surface grows.

## Changes shipped
### Dashboard render-budget telemetry
- Added panel-level render-budget telemetry collection in `dashboard/public/jarvis.html`.
- The dashboard now records per-panel counts for:
  - total items
  - rendered items
  - hidden items
  - estimated DOM-node load
  - whether wallboard virtualization is active
- Extended the existing Layout Budget panel so it reports the active wallboard virtualization preference, how many panels are currently virtualized, and the heaviest panels by estimated render cost.

### Bounded wallboard virtualization
- Added bounded virtualization helpers for wallboard mode with `auto`, `off`, and `on` modes.
- Applied virtualization to the panels most likely to balloon in wallboard mode:
  - news ticker
  - OSINT stream
  - review queue
  - evidence audit
- In virtualized wallboard mode, the client now shows a clear operator-facing note when only a bounded slice is rendered for responsiveness.
- Preserved full rendering outside wallboard mode.

### Settings and contract support
- Extended operator layout settings with `layout.performance.wallboardVirtualization`.
- Exposed the setting through the operator/admin settings contracts and the injected runtime bootstrap state.
- Added admin-surface control coverage so local admin can explicitly set wallboard virtualization mode.
