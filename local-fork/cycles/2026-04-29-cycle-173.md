## Goal
Harden the `restart-safe` helper so listener verification does not hard-fail when `lsof` is unavailable on the host.

## Backlog review and promotions
- Reviewed the already-promoted backlog item for restart-safe helper portability.
- No new unscheduled backlog items were added in this cycle.

## Changes shipped
- Added `resolveLsofPath()` so runtime restart checks try common absolute `lsof` locations before relying on `PATH`.
- Added a `ps`-based owned-process fallback for Crucix listener verification when `lsof` is unavailable.
- Threaded repo-aware listener lookup through `scripts/restart-helper.mjs` so restart validation can use either `lsof` or the fallback path consistently.
- Recorded `listenerLookup` in restart results and retained runtime restart audit entries so operators can see whether verification used `lsof` or fallback mode.
- Extended restart-helper tests with direct coverage for the new `resolveLsofPath()` contract.
