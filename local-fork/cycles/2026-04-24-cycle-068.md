# Crucix dev cycle 068

- Date: 2026-04-24
- Plan item: Epic 21, effort 21.1
- Title: Add first-class operator queue actions such as ack, snooze, and jump-to-artifact links so the review queue becomes directly actionable

## What changed

- Extended `buildOperatorReviewQueue(...)` so each visible queue item now publishes first-class operator actions instead of only descriptive text.
- Added per-item action descriptors for:
  - `ack`
  - `snooze`
  - `artifacts`
- Queue payloads now include compact `ackSummary` visibility so operator surfaces can show how many items are currently dismissed and when the next dismissal expires.
- Added a new API action endpoint:
  - `POST /api/brief/news/review/snooze?region=...&reason=...&hours=24`
- Updated `ackReviewItem(...)` to support explicit action metadata and custom dismissal durations so ack and snooze share one persisted review-dismissal mechanism.
- Enhanced `GET /api/brief/news/review/artifacts` to support filtered artifact jumps by `region` and `reason`.
- Updated the dashboard review queue panel so queue items render clickable action buttons for Ack, Snooze 24h, and Artifacts.
