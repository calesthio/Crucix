# Crucix Dev Cycle 092 - Persisted panel workspace controls

- Date: 2026-04-25
- Plan item: `epic-28 / effort-28.2`
- Status: completed

## Goal
Add practical panel workspace controls so operators can keep high-value windows usable as the dashboard grows, with persisted collapse, pin, priority, and size state instead of fixed panel order.

## Changes shipped
- Extended the operator settings store to persist `preferences.layout.panels` with normalized per-panel state:
  - `collapsed`
  - `pinned`
  - `priority`
  - `size` (`compact`, `normal`, `wide`)
- Exposed `layout.controls.panelPreferences` through the operator and admin settings contracts.
- Preserved panel preferences in the admin settings save flow.
- Added dashboard panel controls in `jarvis.html` for major lower-grid and right-rail panels:
  - pin/unpin
  - collapse/expand
  - priority up/down
  - size cycling
- Added panel sorting so pinned and higher-priority panels render first within the lower and right workspace regions.
- Added panel state styling for pinned, collapsed, and size variants.
