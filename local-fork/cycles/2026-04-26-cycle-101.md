# Crucix Dev Cycle 101 - Source health history and admin source controls

- Date: 2026-04-26
- Plan item: `epic-30 / effort-30.3`
- Status: completed

## Goal
Expose source health history, freshness thresholds, failure-class trends, and one-click quarantine or suppression controls for noisy or broken sources.

## Changes shipped
- Extended `sourceConsole` with source health history metadata:
  - `healthHistory.version = "source-health-history-v1"`
  - current failing, stale, and degraded counts
  - current failure-class counts
  - trend windows sourced from runtime health summaries
  - noisy-source summaries with live state, failure class, freshness target, and live age
- Extended operator source selection persistence with:
  - `preferences.sources.suppressedSourceIds`
  - `preferences.sources.quarantinedSourceIds`
- Extended the operator settings and source console contract with admin source controls:
  - `sourceControls.version = "source-ops-control-v1"`
  - `sourceControls.endpoint = "/api/source-ops/control"`
  - supported actions for suppress, unsuppress, quarantine, and clear-quarantine
- Added a local-admin write endpoint:
  - `POST /api/source-ops/control`
- Updated `/source-ops` to surface:
  - source health window summaries
  - failing-now counts
  - per-source freshness target and live age
  - per-source failure class
  - one-click quarantine and suppression controls with local-admin write boundary preserved
