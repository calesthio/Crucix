# Crucix dev cycle 066

- Date: 2026-04-24
- Plan item: Epic 20, effort 20.2
- Title: Separate per-surface LLM supportability from participation so configured, static-by-design, not-invoked, and unavailable states do not collapse into one operator label

## What changed

- Refined the shared LLM operator-state contract so each surface now distinguishes:
  - `supported`
  - `available`
  - `attempted`
  - `participated`
- Updated analysis surface semantics so a supported-but-unused path reports `not-invoked` instead of being folded into generic deterministic wording.
- Updated ideas surface semantics so `ideasSource: disabled` under a configured runtime now reports:
  - `reason: static-by-design`
  - `label: STATIC BY DESIGN`
  - `supported: true`
  - `available: true`
  - `attempted: false`
  - `participated: false`
- Extended `buildOperatorLlmStateContract()` to publish explicit `support` and `participation` sections alongside the surface details.
- Restarted the local Crucix runtime so the live service serves the refined contract.
