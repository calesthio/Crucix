# Crucix Dev Cycle 155 - Moved dashboard map sizing into explicit layout tokens

- Date: 2026-04-27
- Plan item: `epic-46 / effort-46.1`
- Status: completed

## Goal
Promote the dashboard map height tokenization follow-up into the roadmap and replace scattered inline map height clamps with explicit layout tokens so future dashboard-density tuning is less brittle.

## Changes shipped
- Promoted dashboard map sizing tokenization into `local-fork/plan.json` as `epic-46 / effort-46.1` and completed it in the same cycle.
- Updated `dashboard/public/jarvis.html` CSS:
  - added root layout tokens:
    - `--map-min-height`
    - `--map-height`
  - moved layout-specific map sizing overrides into `body[data-layout-mode=...]` token assignments for `narrow`, `desktop`, and `wallboard`
  - changed `.map-container` to consume the shared tokens instead of hard-coded inline height math
  - updated the compact/mobile rule to consume the same tokens rather than reintroducing separate hard-coded values
- Preserved the already-shipped proportional sizing behavior while making the control surface for future tuning much more obvious.
- Advanced cycle tracking to 155.

## Validation
- Live dashboard verification on `http://127.0.0.1:3117/` after reload showed:
  - layout mode: `desktop`
  - measured map size: about `656x420`
  - active CSS token values:
    - `--map-min-height = 420px`
    - `--map-height = clamp(420px,58vh,680px)`

- This confirmed the tokenized version was live and preserved the corrected proportional footprint.
