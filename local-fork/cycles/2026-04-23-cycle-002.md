# Crucix dev cycle 002

- Date: 2026-04-23
- Objective: complete effort 1.2, OpenSky degraded-state cooldown and fallback hardening
- Status: completed

## Planned work

1. Prevent OpenSky 429s from collapsing the air picture to a guaranteed empty result.
2. Serve a recent-good cached OpenSky snapshot during full live-query failure.
3. Surface degraded state explicitly in source health and air metadata.
4. Restart the local runtime and verify health/data behavior.

## Results

- Added OpenSky local cache support at `runs/cache/opensky-latest.json`.
- OpenSky now persists recent-good hotspot snapshots when live data contains aircraft activity.
- On full live-query failure with a fresh cached snapshot, OpenSky now returns degraded cached data with:
  - `servedFromCache: true`
  - `degraded: true`
  - `stale: true`
  - `cacheAgeMinutes`
  - `liveError`
- Added `degraded` state handling to normalized source health summaries.
- Updated dashboard synthesis so source-provided OpenSky cache fallback is preserved in `airMeta` instead of being treated as an empty live air picture.
- Restarted the local runtime after removing the synthetic validation cache so prod was not polluted by fake air data.

## Evidence

- Direct OpenSky fallback validation:
  - `node apis/sources/opensky.mjs`
  - returned cached/degraded response with `servedFromCache: true` and `liveError: HTTP 429: Too many requests`
- Runtime health after deploy:
  - `curl -fsS http://127.0.0.1:3117/api/health`
  - returned `sourceHealthSummary` including `degraded: 0`, `failed: 5`, proving runtime stayed healthy after restart
- Runtime data after deploy:
  - `/api/data` remained available and showed current OpenSky state plus updated health-summary schema including `degraded`

## Notes

- The live runtime currently has no real cached OpenSky snapshot yet, so OpenSky still reports as failed rather than degraded until the first successful or partially successful live capture with aircraft activity is stored.
- This is intentional. Validation used a temporary synthetic cache file, which was deleted before the production restart.
