# Crucix Cycle 045 - 2026-04-24

## Summary
- Completed Epic 11 effort 11.3: rolling or decayed per-region failure-rate windows so recovering regions stop looking chronically weak.
- Replaced the old raw-failure-only emphasis with decayed failure/success windows, allowing operator surfaces to show when a region is actually recovering.
- Added recovery and decay metadata to persistent review stats instead of forcing operators to infer it from totals.

## What changed
- Added `CLUSTER_REVIEW_DECAY_HALF_LIFE_MS` in `server.mjs` (3-day half-life).
- Extended persistent review-region state with:
  - `failureWindow`
  - `successWindow`
  - `lastWindowAt`
- Updated `recordClusterReviewStats()` to decay prior windows before recording each new observation.
- Updated `summarizeClusterReviewStats()` to expose:
  - `decayedFailureWindow`
  - `decayedSuccessWindow`
  - `decayedFailureRate`
  - `recovering`
  - `recoveringRegionCount`
  - `decayHalfLifeHours`
- Changed top-region ranking to use decayed failure pressure instead of only lifetime failure totals and streaks.
- Added regression coverage in `test/cluster-review-decay.test.mjs`.
