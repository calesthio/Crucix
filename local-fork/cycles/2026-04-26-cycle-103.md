# Crucix Dev Cycle 103 - LLM operations page

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.1`
- Status: completed

## Goal
Create an LLM operations page that exposes provider health, active model, fallback chain, forced heuristic or forced LLM modes, and recent failure reasons.

## Changes shipped
- Added `buildLlmOperationsContract()` in `server.mjs` and exposed it at `GET /api/llm/operations` as `llm-operations-v1`.
- Added a new operator-visible route at `/llm-ops` backed by `dashboard/public/llm-ops.html`.
- The new LLM operations contract now exposes:
  - provider posture, active model, base URL, and configuration-derived health
  - current runtime LLM state via the shared operator LLM status contract
  - forced-mode visibility for `auto`, `off`, and `force`, including explicit forced-heuristic and forced-LLM flags
  - fallback-chain summaries for news clustering, agent analysis, and ideas generation
  - recent failure reasons aggregated from clustering fallback state, analysis refinement errors, ideas failures, and runtime failure telemetry
- Updated adjacent navigation so operator settings, diagnostics, and admin settings link to the new LLM ops surface.
- Extended regression coverage in:
  - `test/settings-contract.test.mjs`
  - `test/settings-endpoint-contract.test.mjs`
- Marked `effort-31.1` complete in `local-fork/plan.json` and advanced Epic 31 to in-progress.
