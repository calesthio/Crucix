# Crucix Dev Cycle 162 - Added operator-editable workspace preset authoring

- Date: 2026-04-28
- Plan item: `epic-36 / effort-36.5`
- Status: completed

## Goal
Replace purely code-defined workspace presets with an operator-editable preset library that can be cloned, saved, imported, exported, and safely persisted through the existing local admin boundary.

## Changes shipped
### Server-side preset persistence
- Added `preferences.layout.customPresets` to the operator settings store and normalization path.
- Extended the shared preset catalog builder so built-in and custom presets are emitted through one contract.
- Runtime HTML bootstrap now injects the merged preset library for dashboard surfaces.

### Settings and admin contracts
- Extended the layout controls contract with:
  - `customPresets`
  - `importExportSupported`
  - per-preset `builtIn` metadata
- Updated layout mutability from `built-in` to `built-in-plus-custom` so the contract tells the truth about editable presets.

### Local admin preset authoring UX
- Reworked `dashboard/public/admin-settings.html` preset helpers so the admin surface can:
  - clone the selected preset into a custom preset
  - save form changes back into a selected custom preset
  - delete a selected custom preset
  - export the selected preset as JSON
  - import a custom preset from JSON
- The preset selector now refreshes against local custom state, so imported or newly cloned presets become selectable immediately before save.
- Durable writes still go through the existing local-admin nonce plus revision token boundary.

### Regression coverage
- Extended contract and persistence coverage for custom presets and editable preset metadata.
- Hardened the served-page smoke test to use a randomized port so a pre-existing local listener does not cause a false pass/fail mix.
