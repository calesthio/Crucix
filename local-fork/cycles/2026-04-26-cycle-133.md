# Crucix Dev Cycle 133 - Added schema-level validation helpers for source-ops artifacts

- Date: 2026-04-26
- Plan item: `epic-34 / effort-34.1`
- Status: completed

## Goal
Add reusable schema validation helpers so source-ops example artifacts are checked against their JSON schemas instead of relying on spot-check assertions alone.

## Changes shipped
- Added `lib/schema-validator.mjs`, a lightweight JSON-schema validator supporting the schema features currently used by Crucix source-ops artifacts.
- Added `lib/source-ops-schemas.mjs` to load and validate source-ops artifacts by schema path.
- Added new lifecycle schemas:
  - `source-ops/schemas/lifecycle-evaluation.schema.json`
  - `source-ops/schemas/lifecycle-batch.schema.json`
- Extended `source-ops/profile.json` with lifecycle schema-path metadata.
- Updated `lib/source-ops-runtime.mjs` and `server.mjs` so the source-console contract exposes the new lifecycle schema paths.
- Strengthened `test/source-ops-contract.test.mjs` to validate example tasks, scorecards, overlap assessments, pruning assessments, and lifecycle evaluation/batch artifacts against their schemas.
- Extended runtime and settings contract coverage to assert the new source-console schema-path metadata is surfaced.
- Updated `local-fork/plan.json` to mark `effort-34.1` complete.
