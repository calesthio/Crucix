## Results

- Added a new backlog item for per-context last-access timestamps and optional LRU-style eviction if production chat traffic needs something smarter than a simple max-entry cap.
- Added bounded selection-memory constants and helpers in `server.mjs`:
  - `SELECTION_MEMORY_MAX_ENTRIES`
  - `pruneSelectionMemory()`
  - `selectionMemoryStats()`
- Selection memory now prunes expired entries opportunistically and evicts oldest entries when the configured max-entry cap is exceeded.
- Extended memory-facing APIs to expose runtime state:
  - `GET /api/brief/context?context=...` now returns `memory` stats
  - `GET /api/brief/context/health` returns aggregate selection-memory health
  - `/api/health` now includes `selectionMemory`
- This makes the short-lived chat-state layer bounded and directly observable from the runtime.
