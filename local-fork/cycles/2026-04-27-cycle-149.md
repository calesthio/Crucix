# Crucix Dev Cycle 149 - Added bounded SDR session plans and retained RF evidence placeholders

- Date: 2026-04-27
- Plan item: `epic-40 / effort-40.1`
- Status: completed

## Goal
Advance SDR corroboration beyond receiver selection by adding bounded session plans, expected capture artifacts, and retained RF-side evidence placeholders that operators can later review, without claiming unattended radio capture or attribution.

## Changes shipped
- Added `SDR_SESSION_STATE_KEY` in `server.mjs`.
- Added SDR session audit helpers:
  - `getSdrSessionState()`
  - `saveSdrSessionState(state)`
  - `buildSdrObservationChecklist(profile)`
  - `buildSdrSessionArtifacts(sessionId)`
  - `reconcileSdrSessionAudit(priorityChecks, nowIso)`
- Extended `buildSdrCorroborationContract(snapshot)` so SDR corroboration now exposes a retained `sessionAutomation` block:
  - `sessionAutomation.version = "sdr-session-audit-v1"`
  - `sessionAutomation.stateKey`
  - `sessionAutomation.updatedAt`
  - `sessionAutomation.sessionCount`
  - `sessionAutomation.recentSessions`
  - `sessionAutomation.statuses`
  - `sessionAutomation.artifactTypes`
- Added bounded per-session retained structure including:
  - `sessionId`
  - `status`
  - `automationMode = "bounded-session-plan"`
  - `watchProfile`
  - `suggestedReceivers`
  - `observationChecklist`
  - `evidenceSummary`
  - `artifacts.expectedArtifacts`
- Upgraded SDR capability posture from receiver-selection only to:
  - `automationMode = "bounded-session-plan-and-evidence-retention"`
  - `artifactsRetained = true`
  - `supportsObservationSummaries = true`
- Preserved the truth boundary:
  - Crucix still does not claim unattended RF decoding, unattended capture, or automatic attribution.
  - Empty evidence remains explicitly represented as `effectOnClaim = "not-yet-observed"`.
- Added regression coverage proving the settings contract exposes planned SDR sessions and retained evidence placeholders.
