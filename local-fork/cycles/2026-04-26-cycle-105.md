# Crucix Dev Cycle 105 - Agent-analysis tuning controls

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.3`
- Status: completed

## Goal
Add agent-analysis tuning controls for tipping-point thresholds, publish policy, deterministic fallback mode, and horizon behavior so output can be tuned without code edits.

## Changes shipped
- Extended operator settings defaults and normalization in `server.mjs` for agent-analysis tuning:
  - `publishPolicy`
  - `deterministicFallbackMode`
  - `horizonBehavior`
  - `tippingPointMinProbability`
  - `maxPublishedTippingPoints`
- Extended published agent-analysis shaping so runtime output now respects tuning for:
  - minimum outlook confidence under strict versus balanced policy
  - allowed horizon scope and maximum published horizon count
  - tipping-point probability threshold and max published tipping points
  - deterministic fallback suppression when policy forbids publishing draft analysis
- Added a fallback-suppressed analysis builder so the runtime stays truthful when policy withholds deterministic draft output.
- Exposed tuning controls through the settings contract under `agentAnalysis.controls`, including available option lists.
- Updated settings surfaces:
  - `dashboard/public/settings.html` now shows publish policy, fallback mode, horizon behavior, tipping threshold, and max tipping points
  - `dashboard/public/admin-settings.html` now allows local-admin editing of those controls
- Expanded regression coverage in:
  - `test/settings-contract.test.mjs`
  - `test/operator-settings-persistence.test.mjs`
