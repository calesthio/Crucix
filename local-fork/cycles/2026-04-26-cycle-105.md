# Crucix Dev Cycle 105 - Agent-analysis tuning controls

- Date: 2026-04-26
- Plan item: `epic-31 / effort-31.3`
- Status: completed

## Goal
Add operator-tunable agent-analysis controls for tipping-point thresholds, publish policy, deterministic fallback mode, and horizon behavior so analysis output can be tuned without code edits.

## Changes shipped
- Extended `operatorSettingsDefaults()` and `normalizeOperatorSettings()` in `server.mjs` with persisted `preferences.agentAnalysis` controls for:
  - `publishPolicy`
  - `deterministicFallbackMode`
  - `horizonBehavior`
  - `tippingPointMinProbability`
  - `maxPublishedTippingPoints`
- Wired the new settings into runtime behavior in `server.mjs` by:
  - centralizing tuning reads through `getAgentAnalysisTuning()`
  - applying publish-policy, horizon, and tipping-point filtering in `buildPublishedAgentAnalysis()` and `dedupePublishedOutlook()`
  - adding policy-driven deterministic-fallback suppression via `buildFallbackSuppressedAgentAnalysis()` and `buildAgentAnalysis()`
  - exposing the active controls through the operator/admin settings contracts instead of leaving them marked as planned
- Expanded `dashboard/public/admin-settings.html` so the local admin surface can edit the new agent-analysis controls directly.
- Expanded `dashboard/public/settings.html` so the read-only operator surface shows the active publish policy, fallback mode, horizon behavior, tipping threshold, and tipping-point cap.
- Extended regression coverage in:
  - `test/settings-contract.test.mjs`
  - `test/operator-settings-persistence.test.mjs`
- Marked `effort-31.3` complete in `local-fork/plan.json` and advanced cycle state to 105.
- Added a follow-on backlog item for endpoint-level regression coverage of settings-driven `/api/analysis` behavior.
