# Crucix dev cycle 079

- Date: 2026-04-25

- Title: Add a lifecycle policy evaluator that consumes candidate records plus scorecard and overlap artifacts and emits the exact next allowed transition or blocking reason

## What changed

- Added `lib/source-lifecycle-policy.mjs` with a bounded lifecycle evaluator that:
  - reads the current lifecycle state
  - checks requirement satisfaction against `source-ops/lifecycle-transition-policy.json`
  - applies promotion-readiness guards
  - computes the next allowed transition candidate
  - records blocking reasons when the next state crosses a human boundary or the current state is not agent-advanceable
- Added an example lifecycle evaluation artifact at:
  - `source-ops/results/onboarding-prep/example-lifecycle-evaluation-maritime-001.json`
- Extended `lib/source-ops-runtime.mjs` so `sourceOps` now exposes:
  - `lifecycleEvaluation`
- The example evaluator path currently shows the canonical shadow candidate as:
  - `currentState = shadow`
  - `promotionReadiness = shadow-ready`
  - `nextAllowedState = approved`
  - blocked because the next step crosses the human approval boundary and shadow is not agent-advanceable
- Expanded validation coverage in:
  - `test/source-ops-contract.test.mjs`
  - `test/source-ops-runtime.test.mjs`
