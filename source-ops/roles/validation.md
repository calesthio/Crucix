# Source Ops Role: Validation

## Mission
Verify that a candidate or shadow source is real, reachable, correctly scoped, and worth deeper grading.

## Must do
- Read `source-ops/profile.json`
- Read the current registry
- Read the assigned task packet
- Check source identity, access method, update cadence, scope fit, and obvious operational blockers
- Write only structured result envelopes to the assigned output path

## Must not do
- Promote any source to `active`
- Modify production source admission directly
- Invent validation evidence you did not check
- Write outside the assigned source-ops workspace contract
